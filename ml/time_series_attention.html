I · Motivation
02 / 14
Why forecasting is different

A useful dependency is often a lag, a scale, or a coupled sensor.

In language, attention often asks which tokens are semantically related. In forecasting, the stronger question is usually: which previous delay, which temporal resolution, or which other channel helps explain the next value?

A weather model may care about yesterday at the same hour. A power-grid model may care about the weekly cycle. A sensor network may care about whether temperature and pressure move together at one frequency but not another.

Why plain full attention is not enough

  • It compares every timestamp with every other timestamp, even when only a few seasonal lags matter.
  • It has no built-in notion of multi-resolution structure.
  • It treats channels implicitly unless we tell it otherwise.
Decomposing a signal into trend, seasonal, and local views

II · Baseline
04 / 14
Start from vanilla attention

Full attention remains a baseline, but it carries weak forecasting bias.

Vanilla self-attention $$\operatorname{Attn}(Q,K,V)=\operatorname{softmax}\left(\frac{QK^\top}{\sqrt{d}}\right)V.$$

This formula says every time point may compare itself with every other time point. That flexibility is attractive, but it does not tell the model that lags or frequencies deserve special treatment.

Forecasting mismatch

The expensive part is not only the $N^2$ cost. The deeper issue is that the model must rediscover seasonality, scale, and channel structure from scratch.

1
Useful when the horizon is short or the sequence behaves more like an event stream than a seasonal process.
2
Weak when long windows are dominated by a small set of recurring lags.
3
Missing bias for channels that lock together only in certain bands.
Time-series attention variants are best understood as biasing mechanisms: they narrow the search to the structures that forecasting data typically exhibits.

II · Positional Encoding
How models know where they are in time

Time-series transformers need richer positional signals than NLP models.

NLP uses simple sinusoidal or learned position indices. In forecasting, position carries additional meaning: time of day, day of week, month, and holiday status are all semantically distinct from a raw index.

Sinusoidal baseline (Vaswani et al.) $$\text{PE}(t, 2i) = \sin\!\left(\frac{t}{10000^{2i/d}}\right), \quad \text{PE}(t, 2i{+}1) = \cos\!\left(\frac{t}{10000^{2i/d}}\right)$$
This only encodes position order, not calendar structure. For a forecasting model, knowing "step 48" is less useful than knowing "Monday 08:00" — temporal stamps encode both.
1
Sinusoidal / learnable index. Simple and fast. Works for short sequences with no calendar structure. Loses meaning if the sequence has gaps.
2
Temporal stamps. Embed hour-of-day, day-of-week, month, and holiday as separate learned embeddings, then sum them. Used in Informer, Autoformer, and PatchTST.
3
Rotary / RoPE. Encodes relative position by rotating query/key vectors. Generalizes better to unseen sequence lengths. Growing adoption in forecasting transformers.

Common mistake

Using position index as a proxy for time when the series has irregular sampling — gaps cause misaligned positional embeddings.

II · Patch Tokenization
PatchTST — tokens as local windows

Dividing the series into patches makes each token semantically richer.

Instead of treating each timestep as a token, PatchTST splits the input into non-overlapping (or slightly overlapping) patches of length $P$. Each patch is linearly projected to a $d$-dimensional embedding.

Patch tokenization $$\mathbf{x} \in \mathbb{R}^{N} \;\longrightarrow\; \mathbf{z} \in \mathbb{R}^{\lfloor N/P \rfloor \times d}, \quad \mathbf{z}_i = W_p \cdot \mathbf{x}_{[iP\,:\,(i{+}1)P]}$$
raw series N steps
→
split into patches of size P
→
project each patch → embedding
→
N/P tokens, full attention
Patch tokens are N/P× cheaper in quadratic attention cost, and each token now carries a local temporal context — making the model's job easier.

III · Multi-scale
05 / 14
Multiple resolutions

Multi-scale temporal attention lets short spikes and long trends live in different branches.

One attention map is often asked to do too much. A more forecasting-friendly design builds several views of the same sequence: one at full resolution, one downsampled, one even coarser, then fuses them.

input sequence
→
downsample at several factors
→
attend inside each branch
→
upsample and fuse
The inductive bias is strong and intuitive: if long-range structure matters, it should survive resampling. Coarse branches are cheaper and often cleaner than raw full-resolution attention.
What each branch sees
  • Scale 1 catches local continuity and bursts.
  • Scale 2 smooths noise and reveals medium cycles.
  • Scale 4 emphasizes broad seasonal drift.

Practical design

Most architectures follow the pattern downsample → attend → upsample → mix. The hard part is not the attention block itself, but choosing resampling and fusion rules that preserve alignment.

Section III · Multi-scale Time-Series Attention
III · Pseudocode
07 / 19
Algorithmic template

Most multi-scale temporal blocks look like downsample, attend, upsample, and mix.

Algorithm · multi-scale temporal attention
1input: sequence x ∈ R^(B×N×d), scales S
2initialize empty outputs list Y
3for each scale s in S:
4  if s = 1, keep x_s ← x
5  else compute x_s ← downsample(x, s)
6  build scale-specific mask or window M_s
7  apply attention y_s ← Attn_s(x_s, M_s)
8  if s > 1, set y_s ← upsample(y_s, N)
9  append y_s to Y
10concatenate all branches along feature dimension
11mix with a projection or gating layer
12return fused representation
Design knobs
  • Pooling type: average, max, stride, learned patching.
  • Attention shape: local, sparse, or full at each scale.
  • Fusion: concatenation, weighted sum, or gated mixing.
What can go wrong

If downsampling destroys phase alignment, the coarse branch may learn a cleaner but misleading signal.

What helps

Residual paths and careful upsampling preserve temporal correspondence between branches.

Notice that the attention rule itself is almost unchanged. The real innovation is architectural: create several easier forecasting views, then let them collaborate.

III · Walkthrough
06 / 14
One branch pyramid in slow motion

Multi-scale attention becomes intuitive when we watch the sequence get simpler as resolution decreases.

Step 1 / 4

Step through the pyramid once. The idea is not exotic: each scale solves a simpler forecasting subproblem.

Step 1 · Fine resolution

Begin with the raw sequence.

The full-resolution branch keeps sharp variations, anomalies, and local continuity. It is the branch most similar to ordinary self-attention.

  • Best for spikes and short transients.
  • Most expensive branch.
  • Most sensitive to noise.
Step 2 · Pool into coarser windows

Downsampling removes some detail but exposes larger rhythm.

Average pooling, striding, or learned patching makes the sequence shorter. The coarse branch no longer chases every local bump.

  • Fewer tokens means cheaper attention.
  • Smoother structure makes long cycles easier to detect.
  • Some fine detail is intentionally sacrificed.
Step 3 · Attend within each scale

Each branch now solves its own temporal problem.

Fine branches learn local continuity, while coarse branches can afford broader receptive fields and longer effective memory.

  • The same attention rule behaves differently by scale.
  • Window size often grows relative to the compressed axis.
  • Scales may be local-only, sparse, or full attention.
Step 4 · Upsample and fuse

Forecasting quality comes from combining the scales well.

The final representation mixes the crispness of the fine branch with the seasonality and stability of the coarse branches.

  • Fusion may be concatenation plus projection.
  • Residual connections help preserve alignment.
  • Poor fusion can erase the benefit of the pyramid.

IV · Spectral view
07 / 14
Frequency-domain attention

Long seasonal structure can be compact in Fourier space.

A daily or weekly cycle may look long-range in time but sparse in the frequency domain. This is why many forecasting models transform the series with an FFT before gating or attending.

Spectral workflow $$x(t)\xrightarrow{\mathrm{FFT}}X(f)\xrightarrow{\text{mix or attend}}\widetilde{X}(f)\xrightarrow{\mathrm{IFFT}}\widetilde{x}(t).$$
The win is not automatic subquadratic complexity. The real gain comes when the spectrum is sparse, compressed, or easier to manipulate than the raw sequence.
Why the spectral view helps
  • Periodic structure becomes concentrated in a few bins.
  • Filtering, denoising, and amplification become geometric.
  • Very long horizons become easier to summarize compactly.
Strong periodic components occupy only a few dominant bins

IV · Spectral algorithm
09 / 19
From waveform to bins and back

Spectral attention usually means FFT, attend over bins, then invert.

Algorithm · attention over Fourier bins
1input: multivariate signal x ∈ R^(B×N×C)
2compute real FFT along time: X_f ← rFFT(x)
3build real-valued features from Re(X_f) and Im(X_f)
4project to queries, keys, values per frequency bin
5score bin-to-bin interactions with scaled dot products
6mix values across bins using softmax weights
7map back to a complex spectrum
8invert with irFFT to recover a time-domain sequence
9return filtered or enhanced forecast features
Time view
many oscillations spread across time
Frequency view
a few dominant bins summarize the cycle
The biggest conceptual mistake is to think “FFT makes things cheap by itself.” It does not. The advantage appears when the spectrum is simpler, sparser, or easier to manipulate than the raw timeline.

IV · Cross-channel
08 / 14
Cross-channel spectral attention

Variables may couple at one rhythm and decouple at another.

Multivariate forecasting introduces a second axis of structure: channels. Temperature, load, pressure, and wind may align strongly at a 24-hour pattern, weakly at 1-hour noise, and not at all elsewhere.

Key idea

Perform an FFT along time, then let channels attend to one another inside each frequency bin. The token identity changes again: here, channels are the entities and the frequency bin is the context.

This is especially useful when physical variables share a periodic driver but respond with different amplitudes or phase offsets.
Coupling strength may depend on the spectral band under study

IV · Channel algorithm
11 / 19
Attention over variables, not timesteps

Cross-channel spectral attention is mostly a tensor transpose plus a different choice of attention axis.

Algorithm · channel attention per frequency bin
1input: signal x ∈ R^(B×N×C)
2compute X_f ← rFFT(x) along the time axis
3for each frequency bin, form channel features from Re and Im
4project those channel features to Q, K, V
5score channel-to-channel interactions inside the same bin
6softmax over channels, not over time indices
7mix values to obtain band-specific coupled features
8optionally invert the spectrum back to time
Do not confuse these

Channels belong to the data. They are temperature, wind, load, pressure, or sensor IDs.

Heads are different

Heads belong to the architecture. They are parallel subspaces used to parameterize attention.

Reading the result

  • A strong entry means two variables interact in this band.
  • A weak entry does not mean global independence.
  • The same pair of variables can couple at one band and separate at another.
This mechanism is a good example of how a very small tensor re-indexing change can encode a major modeling prior.

V · Seasonal lags
09 / 14
Autoformer-style autocorrelation

Replace dense pairwise scores with a search over the strongest delays.

For many series, the useful dependency is not an arbitrary pair of timestamps. It is a small set of recurring delays such as 24, 48, or 168. Autocorrelation attention encodes exactly that belief.

Autocorrelation by FFT $$R_{QK}(\tau)=\operatorname{IFFT}\!\left(\operatorname{FFT}(Q)\cdot\overline{\operatorname{FFT}(K)}\right)_\tau.$$
The convolution theorem turns lag search into an $O(N\log N)$ computation. We then keep only the top few delays rather than learning an unrestricted $N\times N$ map.
Interpretation
  • Each $\tau$ is a candidate seasonal delay.
  • Top-$k$ delays act like the model’s hypothesis set.
  • Values are shifted and mixed according to those delays.

Why this is different from softmax attention

Standard attention scores query-key pairs directly. Here we score delay alignment, which is much closer to the structure of seasonal forecasting.

V · Walkthrough
10 / 14
How lag attention is built

Autocorrelation attention is easier to understand as a four-step seasonal matching loop.

Step 1 / 4

This replaces the dense $N \times N$ score matrix with a much more forecasting-specific sequence of operations.

Step 1 · Build query and key summaries

Collapse feature dimensions so lag structure is easier to compare.

Many implementations summarize the per-head feature channels before computing autocorrelation. The goal is to compare temporal patterns, not every latent dimension separately.

  • Queries and keys become cleaner seasonal traces.
  • Feature averaging reduces noise in lag estimation.
  • The value stream is kept for later mixing.
Step 2 · Compute correlation by FFT

Every candidate delay receives a seasonal alignment score.

FFT-based correlation produces a vector indexed by lag $\tau$. Peaks indicate delays where the query pattern and key pattern align well.

  • Cost becomes $O(N\log N)$ rather than $O(N^2)$.
  • Peaks are interpretable as candidate seasonal delays.
  • The result is a lag score curve, not an attention map yet.
Step 3 · Keep only the top-k delays

The model commits to a small hypothesis set.

Instead of mixing over all lags, the mechanism chooses only a few dominant delays. This is a strong but useful inductive bias for seasonal data.

  • Top-k delays are often stable and interpretable.
  • Softmax is applied over selected lags, not all timestamps.
  • This makes attention explicitly seasonal.
Step 4 · Shift values and mix

Values are rolled by the selected delays and then combined.

The final output is a weighted mixture of value streams shifted by the chosen lags. This is the seasonal analogue of the usual attention-weighted sum.

  • Delay selection and value mixing are separated clearly.
  • The operation stays interpretable end to end.
  • Vectorization is critical for efficient implementation.

VI · Implementation
11 / 14
Vectorized lag mixing

The core systems trick is to gather all shifted value streams in parallel.

Algorithm · vectorized lag mixture
1input: values V, top lags L, lag weights a
2build base indices 0,1,...,N-1
3compute shifted indices (base - L) mod N
4expand V to expose a lag axis
5gather all shifted copies in one batched tensor op
6weight each shifted copy by a
7sum over the lag axis
8return mixed output
Why vectorize

A literal Python loop over lags destroys GPU efficiency. The vectorized version turns the whole seasonal mixture into index arithmetic plus one batched gather.

Concept
Build every shifted value stream at once, then mix them.
Win
Same asymptotic idea as autocorrelation, but with much better hardware utilization.
Tradeoff
The code becomes more index-heavy and less visually obvious than a loop.
In practice, the best forecasting ideas usually need two explanations: the modeling story and the tensor-programming story. Vectorized lag mixing is the latter.

VII · Cost and bias
14 / 19
Complexity is only half the story

The best mechanism is not the cheapest one, but the cheapest one whose bias matches the data.

Asymptotics matter, but only together with the inductive bias
Cheap but wrong

A fast mechanism with the wrong structure prior may miss the signal entirely.

Expensive but blind

Full attention is flexible, but it often spends most of its capacity rediscovering obvious seasonal regularities.

Bias pays off

When the data really is seasonal or multi-scale, the right mechanism buys both sample efficiency and compute efficiency.

Bias can hurt

Strong priors become brittle when regimes shift suddenly or when periodic structure disappears.

This is the design principle to teach explicitly: complexity, interpretability, and inductive bias should be evaluated together, not one at a time.

VIII · Choosing tools
12 / 14
Which bias matches the data?

Different time-series attention mechanisms encode different forecasting assumptions.

Mechanism Bias Shines when Main trade-off
Vanilla full attention Arbitrary timestamp-to-timestamp interaction. Short windows, irregular events, weak periodicity. Quadratic and weakly biased toward seasonality.
Multi-scale temporal Fine and coarse patterns should be handled separately. Mixed local spikes and long seasonal drift. Needs careful resampling and fusion.
Frequency-domain Seasonality is sparse in the spectrum. Long smooth periodic components dominate. Discontinuities are less natural spectrally.
Cross-channel spectral Variables interact differently across bands. Multivariate sensors with band-specific coupling. Channel cost can grow quickly.
Autocorrelation attention A few delays explain most useful dependence. Clear seasonal lags and long horizons. Less flexible under abrupt regime change.
Use lag-aware attention when…

you can name the likely delays before training, even if only approximately.

Use multi-scale attention when…

your signal clearly mixes fast local behavior with slow global structure.

Use channel-aware attention when…

the important information lives in interactions across sensors, not only within one sequence.

IX · Applications

Where these ideas matter

Forecasting gains appear when the architecture matches the physical structure of the data.

Energy and load

Daily and weekly cycles dominate. Lag-aware and spectral models often outperform generic full attention.

Weather and climate

Multi-scale structure is essential because local fluctuations sit on top of slower seasonal and regional dynamics.

Industrial sensors

Cross-channel coupling matters because variables may respond to the same machine state at different frequencies.

Traffic and mobility

Rush-hour repetition and weekend effects make explicit lag bias attractive, especially when horizon length is large.

Physiology and biosignals

Multi-resolution and cross-channel views help disentangle local events, slower rhythms, and synchrony between sensors.

X · Evaluation
Measuring forecasting quality

Metric choice shapes what a model optimizes — and what it silently ignores.

Metric Formula Penalizes
MAE $\tfrac{1}{H}\sum|\hat{y}_t - y_t|$ All errors equally; robust to outliers.
MSE $\tfrac{1}{H}\sum(\hat{y}_t - y_t)^2$ Large errors quadratically; outlier-sensitive.
MAPE $\tfrac{1}{H}\sum\left|\tfrac{\hat{y}_t - y_t}{y_t}\right|$ Relative error; unstable near zero.
MASE MAE / MAE of naïve seasonal lag Beats naïve baseline; scale-free.
CRPS proper scoring for distributions Probabilistic calibration; not point forecasts.
Which metric to use
  • MAE when outliers are real events you want to survive, not dominate.
  • MSE when large errors are catastrophic (e.g., load spikes).
  • MASE when comparing across series with different scales.
  • CRPS when you need calibrated uncertainty, not a point estimate.
Models trained on MSE often look worse on MAE benchmarks — not because they are worse, but because MSE over-suppresses the mean relative to the median. Always report both when possible.

Benchmark caution

Many published comparisons normalize inputs differently. MSE on normalized vs. raw targets can change rankings substantially.

XI · Pitfalls
Where forecasting transformers go wrong

Data leakage, distribution shift, and normalization errors are the most common silent failures.

1
Look-ahead leakage. Using future timestamps in positional embeddings, normalization statistics, or data augmentation causes the model to "see the future" during training. Even subtle leaks — like computing global mean over the full series — can inflate benchmark results by 10–30%.
2
Distribution shift. Forecasting transformers trained on stationary windows often degrade sharply at regime changes (COVID, market crashes, equipment faults). Reversible instance normalization (RevIN) or adaptive normalization helps but does not solve it.
3
Normalization timing. Normalizing the context window before attention is good practice, but the normalization statistics must come from the context only — never the prediction horizon. Many implementations get this wrong.

The naïve baseline problem

For long-horizon forecasting, a simple seasonal naïve model (repeat the last observed period) is surprisingly hard to beat. Always include it as a baseline — if your transformer doesn't beat it convincingly, the inductive bias is probably wrong for your data.

Quick checklist

  • Train / val / test split is strictly chronological — no shuffling.
  • Normalization computed from train set only.
  • No future timestamps in positional or calendar embeddings.
  • Seasonal naïve and persistence baselines included.
  • Evaluation horizon matches deployment horizon.
Most reproducibility failures in time-series forecasting papers trace back to one of these three issues.

XII · Takeaways
14 / 14
Summarizing the design logic

The right forecasting attention mechanism is the one that asks the right structural question.

  • Vanilla attention is the flexible baseline, but it does not privilege lags, scales, or channels.
  • Multi-scale attention says different resolutions deserve different treatment.
  • Spectral attention says periodic structure is often simpler in Fourier space.
  • Autocorrelation attention says a few delays explain most long-horizon dependence.
Forecasting attention is less about inventing a new softmax and more about deciding what the model should compare: times, lags, frequencies, or channels.

Practical recipe

  • Inspect whether the series is seasonal, multi-scale, or multivariate.
  • Pick the bias that matches that structure first.
  • Only then optimize efficiency and implementation details.

Final caution

No inductive bias is free. The stronger the structural prior, the more it helps when correct and the more it can constrain the model when the regime changes.
