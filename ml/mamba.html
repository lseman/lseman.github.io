Sequence modeling

Remembering the distant past

Given a sequence $x_1,x_2,\ldots,x_T$, predict what comes next or summarize the whole:

$$ P(x_{t+1}\mid x_1,\ldots,x_t)\qquad y=f(x_1,\ldots,x_T) $$
The hard part isn’t the next token — it’s carrying relevant information across long gaps.
where sequences come from
  • Words in a sentence
  • Audio samples & time series
  • DNA and genomic sequences
  • Sensor and scientific measurements
Recurrent networks

The original idea: a hidden state

$$ h_t=f(h_{t-1},x_t) $$
good
  • Constant memory
  • Linear in length
bad
  • Vanishing gradients
  • Weak long-range memory
  • Strictly sequential
state is the only channel to the past
Transformers

Attention: let every token look at every other

Project the sequence
$$ \at{Q}=XW_Q,\quad \at{K}=XW_K,\quad V=XW_V $$
Mix by similarity
$$ \at{A}=\operatorname{softmax}\!\Big(\tfrac{\at{Q}\at{K^{\top}}}{\sqrt d}\Big),\quad Y=\at{A}V $$

Every output is a weighted blend of all earlier tokens — no information bottleneck.

Why attention works

Direct access at any distance

“was” can attend straight to “dog,” however many tokens sit between them.

No matter the gap, the path length between two tokens is always one.

A different memory

Enter state space models

Borrowed from control theory: don’t store every token — evolve a hidden state that summarizes the past.

$$ \dot h(t)=\st{A}\,h(t)+\st{B}\,x(t)\qquad y(t)=\st{C}\,h(t) $$
input → state (with its own dynamics) → output
Intuition

The state is a learned, evolving memory

      Input
        ↓
   [ State ]
        ↓
      Output
  • What to remember
  • What to forget
  • How to transform information as it flows
Discretized

On a real sequence, it’s a recurrence

$$ h_t=\st{\bar A}\,h_{t-1}+\st{\bar B}\,x_t \qquad y_t=\st{C}\,h_t $$

This looks just like an RNN. The difference is entirely in the structure of $\st{\bar A},\st{\bar B},\st{C}$.

the key word

Special, principled matrices are what let an SSM hold information over very long ranges — where a plain RNN forgets.

Two computations, one model

Recurrence for inference, convolution for training

As a recurrence
$$ h_t=\st{\bar A}h_{t-1}+\st{\bar B}x_t $$

step-by-step · O(1) per token

As a convolution
$$ y=\st{\bar K}\ast x $$

one global kernel · fully parallel

This dual view — long memory and parallel training — is the idea behind S4 and Mamba.

Before Mamba: S4

S4 made SSMs remember

S4 designed special state matrices that capture very long dependencies — linear cost, strong memory.

  • Linear complexity
  • Long-range memory
but…

On language, S4 still trailed Transformers. Something was missing — and the next slides are about what.

What classical SSMs lack

They treat every token the same

token  →  same update rule
token  →  same update rule
token  →  same update rule

A fixed SSM has no dynamic focus — it can’t decide that one token matters more than another.

language needs focus
"John picked up the ball.
 ... many tokens later ...
 He threw it."

remember John, ball — ignore the filler in between

The core insight

Let the update depend on the input

fixed parameters → input-dependent parameters
$$ \st{A},\,\st{B},\,\st{C}\;\;\longrightarrow\;\; \mb{A(x_t)},\;\mb{B(x_t)},\;\mb{C(x_t)} $$

Now the model decides, per token, what to store, forget, and expose — a selective state space model.

In practice $\st{A}$ stays a fixed structured matrix; it is $\mb{B}$, $\mb{C}$ and the step size $\mb{\Delta}$ that depend on $x_t$ — and $\mb{\Delta}$ makes the effective $\mb{\bar A_t}=\exp(\mb{\Delta_t}\st{A})$ input-dependent too.
Selective memory

Keep the signal, drop the filler

Reading a sentence, the model learns to gate tokens by content:

theandof important fact theand another fact

retain what carries meaning; let the rest pass through.

The Mamba block

A block with no attention

One repeated unit: project, run the selective SSM, gate, project out.

  • No attention
  • No $n\times n$ attention matrix
Stack these blocks and you have a full sequence model — pure SSM, start to finish.
Inside the block

Two branches, multiplied

The input feeds the selective SSM and a parallel gate; their product is the output.

Same philosophy as a gated RNN — the gate decides how much of the SSM’s output to let through.
Why Mamba is fast

Linear, not quadratic

$$ \text{Transformer }O(n^2)\quad\text{vs}\quad \st{\text{Mamba }O(n)} $$

Double the sequence length and the Transformer does 4× the work; Mamba does 2×.

Hardware-aware scan

Selectivity breaks the convolution

Input-dependent parameters mean there is no single fixed kernel anymore — so Mamba computes the recurrence with a specialized parallel scan.

the algorithm
  • Associative parallel scan
  • Kernel-fused on the GPU
what you get
  • Linear complexity
  • Parallel training
  • High GPU utilization

Built for very long sequences

Especially when $n\gg 10^4$ tokens — where attention’s quadratic cost stops being affordable.

Honest limitations

Not a drop-in replacement

  • Less interpretable than an explicit attention map
  • Different, sometimes trickier, optimization behavior
  • A smaller tooling and ecosystem footprint
  • Some reasoning tasks still favor attention

An active, fast-moving research area — not a closed book.

The lineage

One long march toward better memory

RNN→ LSTM→ Transformer→ S4→ Mamba→ Mamba-2

Each step trades for the same three things: more memory, better scaling, friendlier hardware.

Key takeaways

Six things to remember

  • Transformers are powerful but $O(n^2)$
  • State space models scale as $O(n)$
  • Classical SSMs lacked content awareness
  • Mamba adds selective memory: $\mb{A(x_t),B(x_t),C(x_t)}$
  • Strong quality at linear-time inference
  • Mamba-2 bridges attention and SSMs

Mamba keeps an RNN’s linear cost and constant memory — and makes that memory selective enough to rival attention.
