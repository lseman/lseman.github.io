Sequence windows and temporal structure

Before choosing an architecture, the first modeling decision is how to frame the sequence. The same raw signal can be turned into very different learning problems depending on what history the model sees and what future it must predict.

In practical terms, we build supervised examples by slicing a time series into windows. Each training pair has an input window \(X_t\) containing recent observations and a target \(Y_t\) describing the quantity we want to forecast or classify. This framing is often the leading source of errors in sequence modeling: a good model can still fail if the data presentation is wrong.

1

Pick the history window

Decide how much past context the model sees and at what cadence the useful signal actually lives.

2

Name the prediction target

Choose whether the output is one summary label, the next point, or a full future trajectory of length \(H\).

3

Declare extra context

Be explicit about covariates, calendar features, and any known-future inputs available at inference time.

These choices define the supervised operator from past to future. A model can only learn what the framing allows, so explicitly naming the window, horizon, and conditioning variables is a scientific practice.

Architectural lens: sequence learning is mostly about deciding where memory lives. In an RNN, memory lives in the recurrent hidden state. In an encoder-decoder, memory is compressed into a context representation. With attention, memory becomes a dynamic weighted read from the whole history.
Typical Tensor View
\[ X \in \mathbb{R}^{B \times T \times d}, \qquad y \in \mathbb{R}^{B \times H \times m} \]
where \(B\) is batch size, \(T\) is input length, \(d\) is features per step, \(H\) is forecast horizon, and \(m\) is output dimension.
Supervised Window Construction
\[ X_t = [x_{t-T+1},\dots,x_t],\qquad Y_t = [x_{t+1},\dots,x_{t+H}] \]
\[ \hat{f}_\theta:\mathbb{R}^{T \times d}\rightarrow \mathbb{R}^{H \times m} \]
Sequence modeling is therefore the estimation of a parametric map \(\hat{f}_\theta\) from observed history to a future trajectory or summary target.

Three modeling questions should always be made explicit. First, what information is available at prediction time? Second, what temporal scale matters? Third, is the task point prediction, probabilistic prediction, or representation learning? These questions determine whether a short recurrent model is enough or whether we need richer memory mechanisms.

01

Vanilla RNN: the basic recurrent update

A vanilla recurrent neural network reuses the same cell at each timestep. A single set of weights is applied repeatedly through time, so the model is time-invariant and parameter-efficient. The hidden state \(h_t\) is the learned summary of everything seen up to time \(t\).

Each recurrent step has three scientific roles:

  1. Combine input and history: compute the affine pre-activation \(a_t = W_x x_t + W_h h_{t-1} + b_h\).
  2. Update the latent state: apply nonlinearity to obtain the new hidden state \(h_t = \tanh(a_t)\).
  3. Produce an output: read from the current state via the output head \(y_t = W_y h_t + b_y\).

The hidden state plays two connected roles: it is both the memory carrier for the past and the interface to the output layer. That dual role is what makes vanilla RNNs powerful but also what creates a strict compression bottleneck.

Core Recurrence
\[ h_t = \tanh(W_x x_t + W_h h_{t-1} + b_h), \qquad y_t = W_y h_t + b_y \]
Backpropagation Through Time View
\[ \frac{\partial \mathcal{L}}{\partial h_t} = \frac{\partial \mathcal{L}_t}{\partial h_t} + \frac{\partial \mathcal{L}}{\partial h_{t+1}} \frac{\partial h_{t+1}}{\partial h_t} \]
\[ \frac{\partial h_{t+1}}{\partial h_t} = \operatorname{diag}(1-h_{t+1}^2)W_h \]
Products of many Jacobians are the source of vanishing and exploding gradients.

State-Space Reading

A vanilla RNN can be read as a learned dynamical system: the hidden state is the latent state, the recurrence is the transition rule, and the output head is the observation model.

Transition And Readout
\[ h_t = f_\theta(h_{t-1}, x_t), \qquad y_t = g_\theta(h_t) \]

Lag-\(\tau\) Gradient

If the loss arrives at time \(T\), the signal reaching \(h_{T-\tau}\) is multiplied \(\tau\) times by local Jacobians. This is the exact place where long-range memory becomes fragile.

Chain Rule Across Time
\[ \frac{\partial \mathcal{L}}{\partial h_{T-\tau}} = \frac{\partial \mathcal{L}}{\partial h_T} \prod_{k=T-\tau+1}^{T} J_k \]

Key Symbols

x_t observed input at step \(t\)
h_t rolling summary of the past up to step \(t\)
W_h recurrent weight matrix reused at every timestep
J_t local Jacobian controlling how gradients pass through one recurrent step

Unrolling the model through time exposes the architecture: repeated copies of the same cell, each receiving both the current input and the previous hidden state. This is the right mental model for understanding how the network stores information, why memory fades, and how gradients flow backward through time.

Scientific interpretation: the vanilla RNN is a nonlinear state-space model. The hidden state \(h_t\) is a learned latent state, the recurrence is the transition model, and the output head is the observation model. The weakness is that all relevant history must be repeatedly compressed into one state vector under repeated nonlinear transformations.

Forward Recurrence And Shared Weights

The same recurrent cell is reused at each step. Time grows to the right, but the parameter set does not change.

If the task only needs short memory, this repeated compression can be enough. If important evidence is far away, the summary can blur it away.

Backward Pass = Product Of Jacobians

Backpropagation through time walks in the opposite direction. Every step multiplies by another local Jacobian, so stability depends on repeated gain being close to one.

This is why initialization, normalization, clipping, and especially gated carry paths matter so much for long-horizon sequence learning.

Gradient propagation through time is best understood by viewing each step as a local linear map. At time \(k\), the Jacobian \(J_k=\frac{\partial h_k}{\partial h_{k-1}}\) describes how a small change in the previous hidden state affects the next hidden state. The chain rule then produces a product of these local maps, so the backward signal is shaped by the cumulative effect across many timesteps.

  • Local linearization: each update is approximated by a matrix \(J_k\).
  • Backward product: gradients accumulate as \(\prod_{k=t+1}^{T}J_k\).
  • Growth or decay: the dominant eigenvalue magnitude of each Jacobian governs whether the signal is amplified or contracted.

This is an asymptotic, trajectory-dependent argument: in a nonlinear system, the effective Jacobian can vary with the input and hidden state. But the product form is always the source of vanishing and exploding gradients. The interactive plot below makes that spectral argument concrete.

Why do vanilla RNN gradients explode or vanish?

Consider the vanilla RNN

\[ h_t = \phi(W_h h_{t-1} + W_x x_t + b), \qquad y_t = g(h_t), \]
with a scalar loss \(L=L(h_T)\). Backpropagation gives
\[ \frac{\partial L}{\partial h_t} = \frac{\partial L}{\partial h_T}\prod_{k=t+1}^{T} J_k, \qquad J_k := \frac{\partial h_k}{\partial h_{k-1}} = D_k W_h, \]
where \(D_k=\mathrm{diag}\big(\phi'(W_h h_{k-1}+W_x x_k+b)\big)\) is the pointwise activation Jacobian.

Linear Case \((\phi = \mathrm{id})\)

If \(\phi\) is the identity, then \(J_k=W_h\) and the gradient becomes a matrix power:

\[ \frac{\partial L}{\partial h_t} = \frac{\partial L}{\partial h_T}\, W_h^{\,T-t}. \]
\[ \Big\|\tfrac{\partial L}{\partial h_t}\Big\| \le \Big\|\tfrac{\partial L}{\partial h_T}\Big\|\, \big\|W_h^{\,T-t}\big\|. \]
\[ \rho(W_h) = \lim_{n\to\infty}\big\|W_h^{\,n}\big\|^{1/n}. \]

By Gelfand's formula, \(\|W_h^{\,n}\|\) behaves asymptotically like \(C\,\rho(W_h)^{\,n}\) up to polynomial factors when \(W_h\) is defective. So the long-term behavior is controlled by the spectral radius \(\rho(W_h)\).

  • If \(\rho(W_h)<1\), then \(\|W_h^{\,T-t}\|\to 0\) exponentially, so gradients vanish.
  • If \(\rho(W_h)>1\), then \(\|W_h^{\,T-t}\|\to \infty\) exponentially, so gradients explode.
  • If \(\rho(W_h)=1\) and \(W_h\) is diagonalizable with unimodular eigenvalues, such as an orthogonal matrix, norms are preserved up to constants.
Eigen-decomposition view

If \(W_h = Q \Lambda Q^{-1}\) is diagonalizable, then \(W_h^{\,n}=Q\Lambda^n Q^{-1}\). Each component along an eigenvector scales like \(\lambda_i^{\,n}\), so the largest \(|\lambda_i|\) governs growth or decay.

When \(W_h\) is not diagonalizable, Jordan blocks add polynomial factors in \(n\), but the exponential rate is still determined by the spectral radius \(\rho(W_h)\).

Nonlinear Case \((\text{general } \phi)\)

Now the Jacobian is \(J_k = D_k W_h\). For many activations, \(\|D_k\|\le \gamma\), where \(\gamma := \sup_z |\phi'(z)|\). For example, \(\gamma\le 1\) for \(\tanh\), and for ReLU active units the slope is also at most one.

\[ \Big\|\prod_{k=t+1}^{T} J_k\Big\| \le \prod_{k=t+1}^{T}\|D_k\|\,\|W_h\| \le (\gamma \|W_h\|)^{\,T-t}. \]
  • If \(\gamma\|W_h\|<1\), gradients decay exponentially.
  • If \(\gamma\|W_h\|>1\), gradients can blow up exponentially.

A sharper statement uses the per-step Jacobian spectra directly. If \(\rho(J_k)\le \eta_k\), then

\[ \Big\|\tfrac{\partial L}{\partial h_t}\Big\| \le \Big\|\tfrac{\partial L}{\partial h_T}\Big\| \prod_{k=t+1}^{T}\eta_k, \]
so the exponential rate is controlled by \(\limsup_{n\to\infty}\big(\prod_{k=1}^{n}\eta_k\big)^{1/n}\).

When inputs drive \(\phi'\) toward zero, the recurrence enters a saturated regime and the Jacobian product shrinks rapidly. When the effective local Jacobians stay close to one, information can propagate much more stably.

Takeaways

  • Root cause: backpropagation multiplies many Jacobians, so dominant eigenvalues set the exponential growth or decay rate.
  • Stabilizers: orthogonal or identity-scaled \(W_h\), LayerNorm, gradient clipping, and truncated BPTT all help control the effective Jacobian product.
  • Structural fix: LSTM and GRU introduce nearly linear carry paths so effective Jacobians can stay closer to one along memory channels.

Formal Statement And Compact Proof

Theorem (Linear RNN). For \(h_t=W_h h_{t-1}\) and loss \(L(h_T)\),

\[ \frac{\partial L}{\partial h_t}= \frac{\partial L}{\partial h_T}\,W_h^{\,T-t}. \]

Let \(\rho=\rho(W_h)\) be the spectral radius. Then as \(T-t\to\infty\),

\[ \Big\|\tfrac{\partial L}{\partial h_t}\Big\| = \Theta\big(\rho^{\,T-t}\big) \quad\text{up to polynomial factors,} \]

so gradients vanish if \(\rho<1\), explode if \(\rho>1\), and are preserved up to constants when \(W_h\) is diagonalizable with \(\rho=1\).

Proof

(1) Chain rule gives \(\frac{\partial L}{\partial h_t}=\frac{\partial L}{\partial h_T}\,W_h^{\,T-t}\).

(2) By Gelfand's formula, \(\rho=\lim_{n\to\infty}\|W_h^{\,n}\|^{1/n}\) for any operator norm. Therefore, for large \(n\), \(\|W_h^{\,n}\|\) behaves like \(C\,\rho^{\,n}\) up to polynomial factors arising from Jordan blocks.

(3) Combine the chain-rule expression with the asymptotic behavior of \(W_h^{\,n}\) to obtain the stated growth rate. □

Corollary For Nonlinear RNNs

With \(J_k=D_k W_h\) and \(\|D_k\|\le \gamma\),

\[ \Big\|\tfrac{\partial L}{\partial h_t}\Big\| \le \Big\|\tfrac{\partial L}{\partial h_T}\Big\| (\gamma\|W_h\|)^{\,T-t}. \]

So the product of per-step Lipschitz constants determines whether the gradient norm decays or blows up exponentially.

Practical Implications

  • Initialize \(W_h\) orthogonally or near-identity so that \(\rho(W_h)\approx 1\), and keep input scale small enough to avoid immediate saturation.
  • Use normalization on recurrent preactivations when appropriate to keep the activation Jacobians from collapsing.
  • Clip the global gradient norm, often in the range \(1\) to \(5\), and use truncated BPTT for long sequences.
  • Prefer gated recurrent models such as LSTM or GRU when long-horizon memory is central, because gates learn effective eigenvalues near one along memory paths.

Sequence-to-one for forecasting and classification

In sequence-to-one architectures, the recurrent stack reads a whole input sequence but only the final hidden state feeds the prediction head. This pattern is common when the goal is to summarize the recent past into a single decision.

  1. Read the history: process \(x_1,\dots,x_T\) through the recurrent network.
  2. Compress the context: use the last hidden state \(h_T\) as the learned summary of the window.
  3. Predict from the summary: compute \(\hat{y}=g(h_T)\) to produce a scalar, vector, or class score.

For time series, this can mean: use the previous 24 hourly measurements to predict the next hour’s load; compress the last 200 vibration samples into a machine-health score; or detect whether the current window indicates a regime change.

Sequence-to-One Objective
\[ h_T = \operatorname{RNN}(x_1,\dots,x_T),\qquad \hat{y} = g(h_T) \]
\[ \mathcal{L}_{reg} = \frac{1}{N}\sum_{n=1}^N \|\hat{y}^{(n)}-y^{(n)}\|_2^2, \qquad \mathcal{L}_{cls} = -\frac{1}{N}\sum_{n=1}^N \sum_c y_c^{(n)}\log \hat{p}_c^{(n)} \]

From an information-theoretic standpoint, the final state \(h_T\) is a bottleneck representation. The model is asked to discard irrelevant temporal details while preserving exactly the statistics needed for the target. This is powerful when the target is a compressed decision, but risky when many distinct futures are plausible or when relevant evidence appears far back in time.

12 0.68
Interpretation: sequence-to-one is not limited to the “last hidden state” literally. We often attach pooling, attention, or skip connections on top of the recurrent outputs, but the architectural intent is still the same: map a variable-length history into one decision.

In practice, sequence-to-one models are often the right baseline for classification, anomaly detection, quality prediction, and next-step regression. They should usually be the first recurrent model tested before moving to heavier sequence-to-sequence designs.

03

Sequence-to-sequence and encoder-decoder forecasting

When we need a full future trajectory rather than one summary output, encoder-decoder architectures become useful. The encoder absorbs the history, and the decoder generates the forecast horizon one step at a time or in parallel depending on the design.

  1. Encode history: read the observed inputs \(x_1,\dots,x_T\) into a context vector or memory bank.
  2. Initialize the decoder: use that context to seed the decoder’s recurrent state or attention mechanism.
  3. Generate the horizon: produce each forecast step \(\hat{y}_h\), conditioning on past outputs and available future covariates.

This separates the problem of remembering the past from the problem of constructing the future, which is why seq2seq is a powerful pattern for forecasting and translation alike.

Encoder-Decoder Idea
\[ c = \text{Encoder}(x_1,\dots,x_T), \qquad \hat{y}_{1:H} = \text{Decoder}(c) \]
Autoregressive Factorization
\[ p(y_{1:H}\mid x_{1:T}) = \prod_{h=1}^{H} p(y_h \mid y_{1:h-1}, x_{1:T}) \]
Recursive decoders model this conditional factorization explicitly; direct multi-horizon heads approximate the whole map in one shot.
Recurrent Encoder-Decoder States
\[ h_t^{enc}=f_{enc}(x_t,h_{t-1}^{enc}), \qquad c = h_T^{enc} \]
\[ s_h^{dec}=f_{dec}(y_{h-1},s_{h-1}^{dec},c), \qquad \hat{y}_h=g(s_h^{dec}) \]
The encoder summarizes the observed past; the decoder unfolds that summary into future predictions.

The Bottleneck View

A classic encoder-decoder pushes the whole observed history through one context vector, call it c. That makes the wiring clean, but it also forces long inputs into one compressed summary.

This is the architectural reason attention helps: it turns one fixed bottleneck into a step-specific memory lookup.

Teacher Forcing vs Inference Rollout

Training usually feeds the true previous target, while inference must feed the model’s own previous prediction. The mismatch is the source of exposure bias.

Decoder design is not just about architecture depth; it is also about what signal is available at each forecast step during training and deployment.

In time-series forecasting, the decoder can represent the next 6 hours of power demand, the next week of sales, or a multi-step control trajectory. The key design choice is whether the decoder receives only the context vector, previous predictions, known future covariates, or an attention read over the input history.

Scientifically, the main tension is between compression and conditioning. A pure encoder-context model compresses history aggressively, while a decoder with recurrence or attention can condition each forecast step on richer information. Teacher forcing is often used during training to stabilize autoregressive decoders, but it introduces a train-test mismatch if the decoder later consumes its own past predictions at inference time.

Intuition: the encoder is a reader and the decoder is a writer. The encoder reads the history and builds a state that answers “what has happened so far?” The decoder then writes future steps one by one, each time asking “given what I already predicted, what should come next?”

LSTM: explicit memory cell and gated updates

LSTMs were introduced to make long-term information flow easier. Instead of storing everything inside one hidden vector, they separate a persistent cell state from the exposed hidden state and use gates to regulate reading, writing, and forgetting.

LSTM Gates
\[ f_t = \sigma(\cdot),\; i_t = \sigma(\cdot),\; \tilde{c}_t = \tanh(\cdot),\; c_t = f_t \odot c_{t-1} + i_t \odot \tilde{c}_t,\; h_t = o_t \odot \tanh(c_t) \]
Expanded Gate Equations
\[ f_t = \sigma(W_f x_t + U_f h_{t-1} + b_f),\qquad i_t = \sigma(W_i x_t + U_i h_{t-1} + b_i) \]
\[ o_t = \sigma(W_o x_t + U_o h_{t-1} + b_o),\qquad \tilde{c}_t = \tanh(W_c x_t + U_c h_{t-1} + b_c) \]

Carry Path Jacobian

Along the direct memory path, the local Jacobian is approximately diagonal with forget-gate entries. This is the mathematical heart of the LSTM’s stability story.

Memory Highway
\[ \frac{\partial c_t}{\partial c_{t-1}} \approx \operatorname{diag}(f_t) \]

Expose vs Store

The cell state and hidden state are intentionally different. The cell stores long-range information; the hidden state is the filtered public view used by downstream layers.

Filtered Readout
\[ h_t = o_t \odot \tanh(c_t) \]

Gate Roles

f_t how much old memory survives into the next step
i_t how strongly the candidate memory is written
c~_t new candidate content proposed from input and previous hidden state
o_t how much of memory becomes visible as \(h_t\)

The central scientific idea is the additive memory path through \(c_t\). In contrast to a vanilla RNN, where state information is repeatedly transformed multiplicatively through \(W_h\), the LSTM cell supports a more stable gradient route because information can be carried forward by gated addition rather than repeatedly overwritten by one nonlinear state update.

Intuition: the LSTM behaves like a carefully managed notebook. The forget gate erases stale lines, the input gate writes new information, and the output gate decides what part of the notebook is visible to the rest of the network right now.

Interpretive view: the forget gate decides what to erase, the input gate decides what to write, and the output gate decides what part of memory becomes visible to downstream computation. This division of labor is why LSTMs became the canonical gated recurrent model.
05

GRU: a lighter gated recurrent unit

GRUs simplify the LSTM by merging some of the gating logic. They use update and reset gates to decide how much of the old state to keep and how much new information to incorporate. In practice, they often perform competitively with fewer parameters and a simpler update path.

GRU Update
\[ z_t = \sigma(\cdot), \qquad r_t = \sigma(\cdot) \]
\[ \tilde{h}_t = \tanh(W_x x_t + r_t \odot W_h h_{t-1}), \qquad h_t = (1-z_t)\odot h_{t-1} + z_t \odot \tilde{h}_t \]
Expanded GRU Equations
\[ z_t = \sigma(W_z x_t + U_z h_{t-1} + b_z),\qquad r_t = \sigma(W_r x_t + U_r h_{t-1} + b_r) \]
\[ \tilde{h}_t = \tanh(W_h x_t + U_h(r_t \odot h_{t-1}) + b_h) \]

Interpolation Form

The GRU can be read as a residual-style correction to the previous hidden state. The update gate decides how far the new state moves toward the candidate proposal.

Residual View
\[ h_t = h_{t-1} + z_t \odot (\tilde h_t - h_{t-1}) \]

Local Carry Strength

If \(z_t\) is small, the unit mostly keeps its previous state. If \(z_t\) is large, the unit overwrites aggressively with the new candidate.

Carry Coefficient
\[ \frac{\partial h_t}{\partial h_{t-1}} \approx 1 - z_t \]

Gate Roles

z_t blend coefficient between old state and candidate state
r_t how much old-state information enters the candidate computation
h~_t proposed new hidden state after reset-modulated mixing

Adaptive Smoothing View

Because the update gate changes across time and features, a GRU behaves like a learned exponential smoother whose mixing coefficient depends on context.

Time-Varying Blend
\[ h_t = (1-z_t)\odot h_{t-1} + z_t \odot \tilde h_t \]

From a modeling perspective, the GRU trades explicit cell-state structure for a more compact gating scheme. The update gate \(z_t\) interpolates between memory preservation and state refresh; the reset gate \(r_t\) determines how strongly previous state information participates when constructing the candidate update. This makes the GRU easier to optimize and cheaper to parameterize while preserving much of the practical benefit of gating.

Intuition: the GRU behaves like a smart adaptive smoother. The update gate asks whether we should mostly keep the old hidden state or replace it with a new proposal, while the reset gate decides how much of the old state should influence that proposal.

Update Gate As An Interpolation Dial

The update gate moves the new state along the line between the old hidden state and the candidate proposal. Small \(z_t\) means preserve memory; large \(z_t\) means rewrite aggressively.

This is why GRUs often feel intuitive numerically: the update is a learned blend, not a full overwrite at every step.

Reset Gate Shapes The Candidate Proposal

The reset gate does not directly choose the final state. Instead, it decides how much of the previous hidden state participates when building the candidate \(\tilde h_t\).

Reset is about how much past state should be consulted while forming a new proposal, not about whether that proposal will finally be written.

Why GRUs Often Work Well

  • They retain the main benefit of gating with fewer moving parts than LSTM.
  • The interpolation formula makes the update easy to interpret numerically.
  • They are often a strong first recurrent baseline for forecasting tasks.
Rule of thumb: if you want a recurrent baseline that is strong, simple, and relatively efficient, a GRU is often a good first stop. If long-horizon memory control is central, LSTM structure can still be valuable.

Scientifically, you can think of the GRU as a learned exponential smoothing mechanism with nonlinear state transitions: it continuously decides how much of the past should persist and how much of the current input should reshape the latent representation.

06

Attention: self-attention, cross-attention, and multi-head structure

Attention relaxes the bottleneck of a single compressed hidden state. Instead of forcing the model to store all relevant information in one vector, it lets the current prediction read selectively from past hidden states or input positions.

  1. Project representations: inputs are mapped into queries, keys, and values.
  2. Compute compatibility: each query is compared with every key to score how relevant each source position is.
  3. Read the values: the scores are normalized to weights, and the values are combined into a context vector for the current step.

For temporal data, that means the forecast at time \(t+h\) can focus on yesterday’s same hour, the most recent spike, or a particular seasonal pattern. This is especially helpful when relevant context is sparse or nonlocal.

Scaled Dot-Product Attention
\[ \text{Attention}(Q,K,V)=\text{softmax}\left(\frac{QK^\top}{\sqrt{d_k}}\right)V \]
Self-attention uses queries, keys, and values from the same sequence. Cross-attention uses queries from the decoder and keys/values from the encoder.

One Query Row At A Time

Pedagogically, one attention row is the cleanest object. A single query compares itself with every key, normalizes the scores, then forms one context vector.

Per-Query Read
\[ s_{j,i} = \frac{q_j^\top k_i}{\sqrt{d_k}}, \qquad \alpha_{j,i} = \frac{e^{s_{j,i}}}{\sum_{\ell} e^{s_{j,\ell}}} \]
\[ z_j = \sum_i \alpha_{j,i} v_i \]

Tensor Shapes

Shapes matter because they reveal what is being compared: rows of \(Q\) ask questions, rows of \(K\) are addresses, and rows of \(V\) carry the content that gets retrieved.

Matrix View
\[ Q \in \mathbb{R}^{T_q \times d_k}, \quad K \in \mathbb{R}^{T_k \times d_k}, \quad V \in \mathbb{R}^{T_k \times d_v} \]

Why Divide By \(\sqrt{d_k}\)?

Without scaling, dot products grow with dimension and the softmax becomes too sharp too early. The factor \(\sqrt{d_k}\) keeps scores in a numerically teachable range.

Score Scaling
\[ \operatorname{Var}(q^\top k) \propto d_k \quad \Longrightarrow \quad \frac{q^\top k}{\sqrt{d_k}} \text{ is better conditioned} \]

Context Is A Convex Combination

Attention does not invent a value vector from nowhere. It builds a weighted average of the available values, so the output stays tied to what the memory bank already contains.

Weighted Average Constraint
\[ \alpha_{j,i} \ge 0,\qquad \sum_i \alpha_{j,i}=1 \quad\Longrightarrow\quad z_j = \sum_i \alpha_{j,i} v_i \]

Self-Attention

Each timestep can read from other timesteps in the same sequence. In time series, this helps one moment compare itself with earlier spikes, repeated daily patterns, or local neighborhoods.

Cross-Attention

The decoder asks questions about the encoder memory. This is the canonical encoder-decoder pattern: the forecast step queries the encoded history instead of relying only on one context vector.

Multi-Head Attention

Different heads can specialize: one head may track local continuity, another daily seasonality, another rare spikes. Their outputs are concatenated and mixed downstream.

Q, K, V As A Retrieval Pipeline

Queries ask what we need, keys say where it lives, and values carry the content that is actually retrieved into the context vector.

In one sentence: attention first decides where to look, then it averages the values stored at those positions.

Self-Attention vs Cross-Attention

The equations are nearly the same, but the source of the query changes the interpretation completely.

Self-attention is intra-sequence comparison. Cross-attention is a decoder question asked against an encoder memory bank.

Multi-Head = Split, Attend, Concat, Mix

Multi-head attention repeats the same retrieval pattern in several learned subspaces, then fuses those distinct relational views with one output projection.

Heads are useful because they can disagree, then the output projection learns how to combine those disagreements into one useful representation.

Causal Mask = Future Positions Removed Before Softmax

A masked row keeps past and present logits, but future columns are sent to \(-\infty\). After softmax, only allowed columns retain nonzero probability mass.

The mask acts on logits before normalization, which is why blocked future columns disappear completely from the final distribution.

Self-attention block

Self-attention: each timestep compares itself with the same sequence

Self-attention starts from one input sequence \(X \in \mathbb{R}^{T \times d}\) and projects it into three learned views: queries, keys, and values. The important conceptual move is that the same sequence is being used twice: once to ask “what am I looking for?” and once to answer “where in the sequence does that information live?”

  1. Create queries from each timestep’s representation.
  2. Compare queries and keys to produce alignment scores.
  3. Use softmax to turn scores into weights and read the values.
Self-Attention Math
\[ Q = XW_Q,\qquad K = XW_K,\qquad V = XW_V \]
\[ A = \text{softmax}\left(\frac{QK^\top}{\sqrt{d_k}}\right),\qquad Z = AV \]
Row \(A_{t,:}\) is the distribution of attention weights used by timestep \(t\).

Pedagogically, one row of the attention matrix is the key object. It says: for the current timestep \(t\), how much should we read from each other timestep? The diagonal entries represent “look at yourself”; off-diagonal entries represent “borrow context from somewhere else.”

  • For time series, self-attention is a learned similarity search. A timestep can find earlier moments that match in shape, level, season, or anomaly profile.
  • It removes the fixed recurrent path length. A distant dependency can be reached in one weighted lookup instead of being carried step-by-step through hidden-state transitions.
  • It is especially useful when relevant context is nonlocal. Daily seasonality, weekly repeats, and rare spikes are easier to revisit directly than to memorize inside one hidden state.

One caution is that unrestricted self-attention in forecasting can accidentally read the future if the architecture is not masked. In causal settings, we usually constrain each timestep to attend only to positions \(\le t\).

Cross-attention block

Cross-attention: decoder queries read from encoder memory

Cross-attention appears when we separate the model into two roles: an encoder that builds a memory bank from the input history, and a decoder that generates outputs. The decoder no longer relies on one global context vector; instead, each decoder step asks its own question about the encoded past.

Cross-Attention Math
\[ Q = HW_Q^{(dec)},\qquad K = EW_K^{(enc)},\qquad V = EW_V^{(enc)} \]
\[ \text{Context}_j = \sum_{i=1}^{T_{enc}} \alpha_{ji} V_i, \qquad \alpha_{ji} = \text{softmax}_i\left(\frac{q_j^\top k_i}{\sqrt{d_k}}\right) \]
Decoder step \(j\) gets its own context vector, rather than sharing one fixed summary.

This is a major architectural upgrade over the old encoder-decoder bottleneck. In the classical bottleneck setup, all encoder information is squeezed into one vector, call it c. Cross-attention replaces that with a differentiable memory lookup, so the decoder can focus on different parts of the input at different output steps.

  1. Encode the history into keys and values.
  2. Produce a query for the current decoder step.
  3. Retrieve context by scoring the query against the encoder keys and weighting the values.
  • In multi-step forecasting, different horizons need different evidence. Predicting the next step may depend on the latest measurements, while predicting 24 steps ahead may need stronger seasonal cues.
  • Cross-attention is asymmetric by design. The decoder provides the query; the encoder provides the searchable memory through keys and the retrieved content through values.
  • It is easier to interpret than a single context vector. For each future step, we can inspect which parts of the history were emphasized.

A useful mental model is: encoder states are the notes in the notebook, decoder states are the questions, and cross-attention is the act of reading the relevant notes for the current prediction.

Multi-head block

Multi-head attention: several attention mechanisms run in parallel

Multi-head attention does not change the core idea of attention; it changes the representational capacity. Instead of one attention map trying to capture every dependency pattern at once, several heads each compute their own queries, keys, values, and attention matrix.

  1. Project the input into several independent query/key/value spaces.
  2. Run attention in parallel for each head.
  3. Concatenate and mix the head outputs with a final projection.
Multi-Head Construction
\[ \text{head}_m = \text{Attention}(QW_Q^{(m)}, KW_K^{(m)}, VW_V^{(m)}) \]
\[ \text{MultiHead}(Q,K,V) = \text{Concat}(\text{head}_1,\dots,\text{head}_M)W_O \]
Each head learns a different projection space, so similarity is measured in several ways at once.

The pedagogical payoff is huge: one head can specialize in short-range continuity, another in periodic repetition, another in abrupt shocks, and another in slow-moving trend context. The final output fuses all of those views.

  • Multi-head attention is diversity of relational lenses. The same sequence is examined through multiple learned notions of similarity.
  • It reduces representational overload. One single attention map no longer has to simultaneously encode local, seasonal, and event-driven structure.
  • For time series, head specialization is often interpretable. Some heads often end up focusing on near neighbors, others on periodic offsets, and others on rare but important peaks.

The subtle point is that heads are not just copies. Because each head has distinct projection matrices, the same pair of timesteps can look similar to one head and irrelevant to another. That is what gives multi-head attention its expressive power.

Masked self-attention block

Causal masking: how attention avoids peeking into the future

For forecasting and autoregressive decoding, unrestricted self-attention is dangerous: timestep \(t\) could attend to timestep \(t+1\), \(t+2\), or even the full future horizon during training. That would leak target information into the representation and make the model unrealistically strong in training but unusable at inference time.

  1. Create raw scores with \(QK^\top/\sqrt{d_k}\).
  2. Add the causal mask so future positions receive \(-\infty\) before softmax.
  3. Normalize and use the resulting weights to read the permitted past values.
Masked Attention Math
\[ S = \frac{QK^\top}{\sqrt{d_k}} + M \]
\[ M_{ij} = \begin{cases} 0, & j \le i \\ -\infty, & j > i \end{cases} \qquad\Longrightarrow\qquad A = \text{softmax}(S) \]
The upper-triangular future positions are suppressed before the softmax, so they get zero attention.

The mask is not a small implementation detail; it is the mechanism that enforces the forecasting rule: the representation at time \(t\) may depend on the past and present, but never on the future. In matrix form, this produces the familiar lower-triangular visibility pattern.

  • Masked self-attention is the attention analogue of causal recurrence. An RNN is causal because information flows left-to-right through time; masked attention is causal because illegal future connections are explicitly removed.
  • Training and inference become aligned. The model learns under the same information constraints it will face at prediction time.
  • This is essential in decoder-only generation. Language modeling, autoregressive forecasting, and iterative horizon generation all rely on masked self-attention inside the decoder.
  • Cross-attention usually stays unmasked in the encoder-decoder setting. The decoder may read any encoder memory position because the encoder sequence is already part of the observed input, not the future target.

A useful teaching contrast is this: self-attention answers “which positions in my own sequence may I look at?”, while causal masking adds the rule “only those that would have been available at that moment in time.” That rule is what makes attention safe for forecasting.

Architecture pattern library

These are the common recurrent design patterns you’ll reuse. The important idea is not the cell by itself, but how it is wired into the task: where inputs enter, where outputs leave, and whether the model needs a compressed summary or direct access to earlier steps.

  1. Define the task wiring: one output, one output per step, or a generated output sequence?
  2. Choose the temporal operator: plain recurrence, gated recurrence, attention, or transformer-style interaction.
  3. Decide whether direct memory access is needed: do you need a bottleneck, or should the model be able to revisit past states directly?
Pedagogical principle: most architecture names mix two distinct ideas: the temporal operator used inside the model, and the input-output wiring pattern used by the task. A GRU can be used in many-to-one, aligned many-to-many, or encoder-decoder form; attention can augment several of those patterns without replacing them entirely.
many-to-one

History to a single target

Typical for next-step forecasting, classification, anomaly scoring, or quality prediction from a temporal window.

many-to-many aligned

One output per timestep

Useful for online tagging, denoising, streaming control, or per-step health estimation from sensor data.

encoder-decoder

Compress then generate

Good when input and output lengths differ or when future generation has its own sequential logic.

attention-augmented

Read from past states directly

Better when the model should revisit specific historical positions instead of pushing everything through a single recurrent bottleneck.

When teaching or designing sequence systems, it helps to separate these levels explicitly. First choose the task map: one label, one next value, or a whole future trajectory. Then choose the temporal mechanism: plain recurrence, gated recurrence, attention, or transformer-style processing. This separation makes architecture comparison much clearer and prevents confusing task format with memory mechanism.

RNN vs LSTM vs GRU vs Attention vs Transformer

This section gives the comparative view across the main sequence-modeling families. The goal is to make the memory mechanism explicit: what is being carried forward, what can be revisited directly, and what kind of temporal dependency each architecture handles naturally.

  1. Ask what kind of memory the task needs: short-term summary, long-range dependence, or sparse lookup?
  2. Ask how predictions will access past information: through a hidden state, gated cell state, or direct attention weights?
  3. Ask how much compute and parallelism the problem allows: low-cost sequential recurrence or higher-cost attention?
Unifying question: every sequence model needs an answer to the same problem: how does the current prediction access useful information from the past? Recurrent models answer with a hidden state, gated models answer with controlled memory, attention answers with weighted lookup, and transformers answer with repeated layers of attention-based interaction.

A second scientific comparison axis is computational. Recurrent models process information sequentially across time and therefore have strong streaming behavior but limited parallelism. Attention-based models expose many more direct temporal interactions and often learn richer long-context structure, but they pay for that capacity with greater computational and memory cost.

Start Here

Use a vanilla RNN when the lesson is really about recurrence itself: unrolling, hidden state, and why gradients struggle over time.

Strong Recurrent Baselines

GRU is often the clean practical default; LSTM is the next step when explicit memory control matters more than cell simplicity.

Modern Long-Context Modeling

Attention and transformers become more attractive when direct lookup, long-range dependencies, and rich contextual interactions matter more than recurrent state compression.

There is no universally best family. The right choice is determined by context length, training budget, online constraints, data volume, covariate richness, and the scientific question being asked of the model.

09

Design trade-offs and model selection

Recurrent architectures are still valuable whenever the sequence is moderate in length, streaming inference matters, or the inductive bias of stateful temporal compression is desirable. The right choice depends on how much memory, interpretability, and computational budget the problem demands.

1

Measure the horizon

Ask whether the useful evidence is nearby, slowly drifting, or only recoverable from sparse long-range events.

2

Judge the data regime

Small and medium datasets often benefit from recurrent inductive bias before paying for attention-heavy flexibility.

3

Respect deployment limits

Streaming systems favor sequential recurrence; offline settings can spend more compute on richer memory access.

Bias-Complexity View
\[ \text{generalization error} \approx \text{approximation error} + \text{estimation error} + \text{optimization error} \]
Simpler recurrent models often have higher inductive bias but lower estimation burden; richer attention models reduce approximation bias at the cost of more parameters and optimization difficulty.

Choose vanilla RNN when

  • The dependency horizon is short and you want the simplest recurrent baseline.
  • The objective is pedagogical and you want to teach recurrence, hidden state, and BPTT clearly.
  • You care about lightweight recurrence more than long-memory robustness.

Choose LSTM when

  • Longer temporal context matters and forgetting must be regulated explicitly.
  • The series mixes scales, such as short oscillations plus slow structural drift.
  • Training stability is a stronger concern than cell simplicity.

Choose GRU when

  • You want a strong recurrent default with fewer gates and fewer parameters.
  • The dataset is moderate and you want a practical balance of speed and memory.
  • You need recurrence but not the full LSTM cell-state machinery.

Add attention when

  • Relevant context is sparse or far away in the input history.
  • You want interpretability via visible temporal weighting patterns.
  • The encoder bottleneck hurts sequence-to-sequence quality or multi-horizon forecasts.
Scientific workflow suggestion: start with the simplest model whose assumptions match the task. Increase architectural complexity only when failure analysis indicates a specific limitation: weak long memory, poor multi-horizon conditioning, inability to retrieve sparse events, or lack of flexibility under rich covariates.

Practical recipe: start with a GRU for a clean recurrent baseline, move to LSTM if long memory seems limiting, and add attention when the forecast needs to selectively revisit the past instead of compressing it into one vector.
