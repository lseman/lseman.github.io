Lab Exercise: Implement Tournament Sort in Python

Before we code, let‚Äôs build intuition: tournament sort behaves like a knockout bracket for numbers. We first run a ‚Äúchampionship‚Äù to find the global minimum, then only replay the matches that were on the winner‚Äôs path.

What is ‚Äútournament‚Äù selection?

  • Players = array values. Pair them up, smaller value wins and moves up.
  • Matches = min comparisons. Each internal node stores (min, winning-leaf-index).
  • Champion = global minimum. It sits at the root after the build.
  • Rematch only the path. After removing the champion (set its leaf to +‚àû), only recompute along that leaf‚Üíroot path.

Illustrated Guide

We implement the classic tournament sorting scheme ‚Äî identical to a selection tree / loser tree variant ‚Äî in explicit micro-steps.

1) BUILD (bottom-up mins with argmin)

Pad to a power-of-two leaf count with +‚àû. Each internal node stores (value, leafIndex) as the min of its two children (stability via left-bias on ties).

Pseudocode 
function BUILD(A):
  L ‚Üê next_power_of_two(|A|)
  leaves[0..L-1] ‚Üê A padded with +‚àû
  for i in 0..L-1:
    T[leaf(i)] ‚Üê (leaves[i], i)
  for level from bottom-1 downto root:
    for each node u with children (l, r):
      T[u] ‚Üê min_pair(T[l], T[r])   // tie ‚Üí pick left child
  • Work: O(n).
  • Why: enables O(log n) per extract.

2) EXTRACT (pop root ‚Üí invalidate leaf ‚Üí path update)

Root holds the global min. Emit it to the sorted prefix, set the winning leaf to +‚àû, then recompute only the nodes along that leaf-to-root path.

Pseudocode 
procedure EXTRACT_ONE(T):
  (val, leaf) ‚Üê T[root]
  output.append(val)
  T[leaf_node(leaf)] ‚Üê (+‚àû, leaf)
  u ‚Üê parent(leaf_node)
  while u exists:
    T[u] ‚Üê min_pair(T[left(u)], T[right(u)])
    u ‚Üê parent(u)
  • Each extract is O(log n) compares/updates.
  • After k extracts, the output holds the sorted prefix of size k.

3) Stability & duplicates

On ties, we prefer the left child (smaller leaf index). This keeps the tournament stable with respect to equal keys.

Binary Tree Mathematics

A complete binary tree can be stored in a flat array using simple index arithmetic. With zero-based indices, the mapping is:

  • Children of a node at index \(i\): \[ \text{left}(i) = 2i + 1,\qquad \text{right}(i) = 2i + 2. \]
  • Parent of a node at index \(j>0\): \[ \text{parent}(j) = \left\lfloor \frac{j-1}{2} \right\rfloor . \]

Why do we multiply by 2?

Each level of a complete binary tree has twice as many nodes as the previous level. When moving from a node \(i\) down one level, there are \(2\) new slots ‚Äúopened,‚Äù so the children start at \(2i\) plus an offset: \(+1\) for the left child and \(+2\) for the right.

Level index ranges. If level \(\ell\) (root is \(\ell=0\)) has \(2^\ell\) nodes, then those nodes occupy the array indices: \[ \big[\, 2^\ell - 1,\; 2^{\ell+1} - 2 \,\big]. \] For example, \(\ell=0\) ‚Üí \([0,0]\), \(\ell=1\) ‚Üí \([1,2]\), \(\ell=2\) ‚Üí \([3,6]\).
Leaf band & padding. If a tree has \(L\) leaves, total nodes are \(2L-1\). Leaves occupy indices \([\,2L-1 - L,\; 2L-2\,] = [\,L-1,\; 2L-2\,]\). When \(n\) (array length) is not a power of two, we pad leaves with \(+\infty\) to reach \(L=\text{nextPow2}(n)\).
