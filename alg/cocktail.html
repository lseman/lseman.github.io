A bidirectional variant of Bubble Sort. Each round makes a forward pass (bubbling the maximum to the end) and a backward pass (bubbling the minimum to the start), shrinking the unsorted window from both sides.

Pseudocode


01start ← 0; end ← n-1
02while start < end:
03  swapped ← false
04  for j = start .. end-1:
05    if A[j] > A[j+1]: swap(A[j], A[j+1]); swapped ← true
06  end ← end - 1
07  if not swapped: break
08  swapped ← false
09  for j = end-1 .. start:
10    if A[j] > A[j+1]: swap(A[j], A[j+1]); swapped ← true
11  start ← start + 1

How It Works

Forward pass moves the largest element to the end of the window; backward pass moves the smallest to the start. The window [start..end] shrinks from both sides.

  • Amber: currently compared pair.
  • Rose: a swap occurred.
  • Emerald: outside the active window (already sorted head/tail).

Complexity

Time
  • Worst: O(n²)
  • Average: O(n²)
  • Best: O(n) if early-exit triggers
Space

O(1) extra memory (in-place)

Like Bubble Sort but can reduce passes when small elements start near the end or large elements near the start.
