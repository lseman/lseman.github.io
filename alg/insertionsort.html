Insertion Sort grows a sorted prefix [0..i] by inserting the key A[i] into its correct place among the previous elements, shifting larger items to the right.

Pseudocode


01for i = 1 .. n-1:
02  key ← A[i]
03  j ← i - 1
04  while j ≥ 0 and A[j] > key:
05    A[j+1] ← A[j]     // shift right
06    j ← j - 1
07  A[j+1] ← key        // insert

How Insertion Sort Works

At pass i, take key = A[i] (highlighted in cyan) and scan left with j (amber), shifting larger elements one position to the right (rose), until the correct spot is found. Then insert the key. The sorted prefix is shown in emerald.

Complexity

Time
  • Worst: O(n²)
  • Average: O(n²)
  • Best: O(n) (already sorted)
Space

O(1) extra memory (in-place)

Comparisons and shifts depend on initial order: nearly-sorted arrays are very fast (few shifts).

Experiments

  1. Try increasing Size and observe how the number of shifts grows on random arrays.
  2. Shuffle until you get duplicates — Insertion Sort is stable (equal keys keep order).
  3. Compare with Selection Sort: Insertion often does fewer comparisons on partially sorted data.
