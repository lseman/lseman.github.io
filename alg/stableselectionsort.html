Stable Selection Sort — Interactive Visualizer

Stable selection sort finds the minimum in the unsorted suffix and inserts it at position i by shifting elements (instead of swapping). Equal keys keep their original relative order.

Pseudocode (Stable)


01for i = 0 .. n-2:
02  minIdx ← i
03  for j = i+1 .. n-1:
04    if A[j] < A[minIdx]:
05      minIdx ← j
06  x ← A[minIdx]
07  while minIdx > i:
08    A[minIdx] ← A[minIdx - 1]   // shift right
09    minIdx ← minIdx - 1
10  A[i] ← x                     // insert

By shifting instead of swapping, equal keys never leapfrog each other → the algorithm is stable.

How Stable Selection Sort Works

At pass i, find the minimum in [i..n-1], store it, shift the block A[i..minIdx-1] to the right by one, and insert the stored minimum at i.

Complexity

Time
  • Worst: O(n²)
  • Average: O(n²)
  • Best: O(n²) (no early exit)
Space

O(1) extra memory (in-place)

Comparisons still total ~n(n−1)/2. Moves count shifts + final inserts; in the worst case a pass can shift ~i elements.
