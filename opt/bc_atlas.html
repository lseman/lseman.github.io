Mode: sequential focus How this architecture works
Node dispatch One driver owns the frontier, processes one node at a time, and decides immediately whether to prune, cut, or branch.
Cut sharing Separators write into one in-process pool, so every cut is reusable on the very next node without synchronization.
Bound propagation Incumbent and dual bound update in one memory image, so pruning decisions always see the freshest state.
Best fit Best first implementation for debugging, profiling, and deterministic experiments.

Auto walkthrough

One node enters the worker with a branching path and basis hint. The driver tries the cheapest tightening passes first, spends LP time only once the node still looks alive, then either extracts an incumbent, strengthens the relaxation, or splits the node into children.

01

Presolve

The node inherits local bounds from its branch path. Cheap reductions remove fixed variables, tighten coefficients, and detect obvious infeasibility before any serious solve begins.

shrink the node
02

LP Solve

The LP engine warm-starts from the parent basis, reoptimizes under the new bounds, and produces the node's current dual bound plus a fractional relaxation point.

produce a bound
03

Separation

If the LP point violates known structure, separators generate cuts. Useful cuts are filtered, added back to the node LP, and the relaxation is solved again only if the node still looks promising.

tighten the relaxation
04

Heuristics

The fractional point is mined for primal candidates through rounding, diving, RINS, or polishing. A new incumbent updates the global cutoff and can prune other open nodes immediately.

hunt for feasibility
05

Branching

If the node remains fractional and worth exploring, the brancher picks a disjunction, creates child bounds, and pushes those children back into the node pool with updated scores and basis hints.

split into children
