What this tableau tells you
WHY Why choose the column with the most-negative reduced cost? ▸

The reduced cost $\bar{c}_j$ measures how much the objective value changes per unit increase in $x_j$, holding all other non-basic variables at zero. A negative $\bar{c}_j$ means increasing $x_j$ will improve the objective — so any column with $\bar{c}_j < 0$ is a valid entering variable.

The most-negative rule (Dantzig's rule) picks the steepest rate of improvement, giving the largest gain per pivot. It is a greedy heuristic: fast in practice, but it does not minimise the total number of pivots.

Alternative — Bland's rule: always pick the smallest subscript $j$ with $\bar{c}_j < 0$. Slower per iteration on average, but provably prevents cycling in degenerate problems. Toggle the rule above to see the difference.

WHY Why use the minimum ratio test to pick the leaving row? ▸

When the entering variable $x_j$ increases by $\Delta$ from zero, basic variable $x_{B_i}$ decreases by $a_{ij}\Delta$. The first basic variable to hit zero limits $\Delta$: $$\theta^* = \min_i\!\left\{\frac{b_i}{a_{ij}} : a_{ij} > 0\right\}$$

We must stop at $\theta^*$ — going further would push $x_{B_i}$ negative, violating the non-negativity constraint and leaving the feasible region. The row achieving the minimum is the most-constrained basic variable, so it leaves the basis.

Rows with $a_{ij} \le 0$ are skipped: increasing $x_j$ does not decrease those basic variables, so they impose no limit. If all entries are $\le 0$, no row limits the increase — the problem is unbounded in that direction.
