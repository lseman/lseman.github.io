Presolve as a learning path

From obvious simplifications to structural reasoning, one safe reduction at a time.

Read top to bottom as a pass through the model. Each rule either proves infeasibility, tightens the feasible region without changing integer optima, removes redundant structure, or records enough information for postsolve to rebuild a solution in the original variables.

1. Clean the easy facts Fixed variables, dominated rows, and activity intervals expose contradictions early.
2. Propagate consequences Bounds, integer divisibility, binary packing, and CG rounding shrink the relaxation.
3. Repeat to fixed point Every change can unlock another rule, so the pass loops until no rule changes the model.
