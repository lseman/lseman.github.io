Motivating Example — A Tiny Production Planning LP

Before the abstractions, keep one concrete model in mind. Suppose a factory produces two products, $x_1$ and $x_2$, using two limited resources.

$$\begin{aligned} \max \quad & 3x_1+5x_2\\ \text{s.t.}\quad & x_1+2x_2\le 8,\\ & 3x_1+2x_2\le 12,\\ & x_1,x_2\ge 0. \end{aligned}$$
  1. Geometry: the feasible region is a polygon in the plane, so the optimum will occur at a vertex.
  2. Standard form: introduce slacks $s_1,s_2\ge 0$ to get equalities.
  3. Simplex view: move from one basic feasible solution to another until no improving reduced cost remains.
  4. Duality view: the optimal dual variables will price the two resources.
  5. Sensitivity view: those dual prices tell us how the optimum changes when the resource limits 8 and 12 change slightly.

This example is intentionally small enough to visualize, but it already contains every major idea developed later: vertices, pivots, dual prices, complementary slackness, and sensitivity.

§01 Problem Formulation

A linear program optimizes a linear function over a polyhedron. We work exclusively in standard form, obtained by introducing slack variables to convert inequalities into equalities:

(P) $$\begin{aligned} \min_{x \in \mathbb{R}^n} \quad & c^\top x \\ \text{subject to} \quad & Ax = b,\\ & x \geq 0, \end{aligned}$$

where $c \in \mathbb{R}^n$, $A \in \mathbb{R}^{m \times n}$, $b \in \mathbb{R}^m$, and $\operatorname{rank}(A) = m \leq n$. For the simple case $Ax \leq b,\;x\ge 0$, convert to standard form step by step:

  1. Introduce slack variables $s \in \mathbb{R}^m_{\ge 0}$ so each inequality becomes equality: $Ax + s = b$.
  2. Stack variables into one vector: $\tilde{x} := \begin{bmatrix}x\\ s\end{bmatrix}$.
  3. Stack columns into one matrix: $\tilde{A} := [A\; I_m]$ so $\tilde{A}\tilde{x}=b$.
  4. Extend costs with zeros on slacks: $\tilde{c} := \begin{bmatrix}c\\ 0\end{bmatrix}$.
$$\begin{aligned} \min_{\tilde{x}\ge 0}\;&\tilde{c}^\top \tilde{x}\\ \text{s.t.}\;&\tilde{A}\tilde{x}=b,\\ &\tilde{A}=[A\;I_m],\quad \tilde{x}=\begin{bmatrix}x\\s\end{bmatrix},\quad \tilde{c}=\begin{bmatrix}c\\0\end{bmatrix}. \end{aligned}$$

Unified Conversion to Standard Form

Real LPs often contain mixed constraint types, free variables, and maximization objectives. The standard-form target stays the same:

$$\min \ \tilde c^\top \tilde x \qquad\text{s.t.}\qquad \tilde A \tilde x = \tilde b,\qquad \tilde x \ge 0.$$

Practical Conversion Checklist
  1. Flip the objective if the original model is a maximization problem.
  2. Normalize constraint directions row by row into equalities.
  3. Split every free variable into the difference of two nonnegative variables.
  4. Introduce artificial variables only where a feasible starting basis is missing.
  5. After this preprocessing, apply Phase I if no primal-feasible basis is obvious.
Definition 1.1 — Feasible Set

The feasible set is the convex polyhedron $\mathcal{P} = \{x \in \mathbb{R}^n : Ax = b,\, x \geq 0\}$. The LP is feasible if $\mathcal{P} \neq \emptyset$, bounded if $c^\top x$ is bounded below on $\mathcal{P}$, and optimal if a minimizer exists.

Quick Check — Problem Formulation

Question: What is the purpose of slack variables when converting $Ax \le b$ to standard form?

Answer: They convert inequalities into equalities, so the LP can be written as $\tilde A\tilde x=b$ with $\tilde x\ge 0$.

§02 Geometry & Basic Feasible Solutions

The key insight of the Simplex method is that the optimum, when it exists, always occurs at a vertex of the polyhedron $\mathcal{P}$. Vertices are algebraically characterized as basic feasible solutions.

Definition 2.1 — Basis and BFS

A basis is an index set $\mathcal{B} \subset \{1,\ldots,n\}$ with $|\mathcal{B}|=m$ such that $B := A_\mathcal{B} \in \mathbb{R}^{m\times m}$ is invertible. To construct the corresponding basic solution:

  1. Partition columns: $A=[B\;N]$ and variables: $x=(x_\mathcal{B},x_\mathcal{N})$.
  2. Write constraints: $Bx_\mathcal{B}+Nx_\mathcal{N}=b$.
  3. Set nonbasic variables to zero: $x_\mathcal{N}=0$.
  4. Isolate basic variables: $Bx_\mathcal{B}=b \Rightarrow x_\mathcal{B}=B^{-1}b$.
$$x_\mathcal{B} = B^{-1}b,\qquad x_\mathcal{N} = 0,\qquad \mathcal{N} = \{1,\ldots,n\}\setminus\mathcal{B}.$$

If additionally $x_\mathcal{B} \geq 0$, the solution is a basic feasible solution (BFS). A BFS is degenerate if some $x_i = 0$ for $i \in \mathcal{B}$.

Theorem 2.2 — Fundamental Theorem of LP

If $\mathcal{P} \neq \emptyset$ and the LP is bounded, then there exists an optimal solution at a vertex of $\mathcal{P}$. Every vertex of $\mathcal{P}$ corresponds to at least one BFS. The total number of BFS is at most $\binom{n}{m}$.

Proof Sketch of Theorem 2.2
  1. The feasible set $\mathcal P=\{x:Ax=b,\ x\ge 0\}$ is a convex polyhedron.
  2. Any feasible point that is not extreme can be written as a convex combination of other feasible points. Repeating this decomposition reduces the point to a convex combination of extreme points; in finite-dimensional LP, these extreme points are exactly the BFS.
  3. Equivalently, by Caratheodory-style reasoning inside the affine space $Ax=b$, every feasible point can be expressed using finitely many extreme feasible points.
  4. If $x=\sum_k \lambda_k v^{(k)}$ with $\lambda_k\ge 0$, $\sum_k \lambda_k=1$, then linearity of the objective gives $$c^\top x=\sum_k \lambda_k c^\top v^{(k)}.$$
  5. A convex combination cannot be strictly smaller than all of its components, so the minimum is attained at at least one extreme point.
  6. Since extreme points of $\mathcal P$ correspond to BFS, there exists an optimal BFS whenever the LP is feasible and bounded.
Recession Cone and Extreme Rays

Unboundedness is not a mysterious failure mode: it is encoded by the directions along which the polyhedron extends to infinity.

$$\operatorname{rec}(\mathcal P):=\{d\in\mathbb R^n: A d = 0,\ d\ge 0\}.$$
  1. If $x\in\mathcal P$ and $d\in\operatorname{rec}(\mathcal P)$, then $x+t d\in\mathcal P$ for every $t\ge 0$.
  2. Such a direction $d$ is a feasible ray of the polyhedron. An recession cone direction can be refined to an extreme ray, which is a recession direction that cannot itself be decomposed into two distinct recession directions.
  3. If there exists $d\in\operatorname{rec}(\mathcal P)$ with $c^\top d<0$ for a minimization problem, then $$c^\top(x+t d)=c^\top x+t\,c^\top d\to -\infty,$$ so the LP is unbounded.
  4. If every recession direction satisfies $c^\top d\ge 0$, then the objective cannot keep improving along rays, so any optimum must come from the extreme-point structure instead.

The arrows trace a typical Simplex traversal: starting from the origin, moving along edges of $\mathcal{P}$ to adjacent vertices with improving $c^\top x$, until the optimal vertex $x^*$ is reached.

Geometry Insights — Bounded, Infeasible, and Unbounded LPs

Three geometric pathologies determine the global behavior of a linear program: a bounded feasible polyhedron gives a vertex optimum, incompatible constraints give no feasible point, and an open descent direction gives no finite optimum.

  1. Bounded feasible LP: the best objective contour eventually touches the polyhedron at a vertex, which is why a BFS can be optimal.
  2. Infeasible LP: the half-spaces never intersect, so there is no point satisfying all constraints simultaneously.
  3. Unbounded LP: there exists a feasible ray along which the objective keeps improving, so no finite optimum exists.
Quick Check — Geometry

Question: If the feasible set is nonempty and the objective keeps decreasing along a feasible ray, what should you conclude?

Answer: The LP is unbounded. There is no finite optimum because the objective can keep improving without leaving the feasible set.

§03 Reduced Costs & Optimality

At a BFS with basis $\mathcal{B}$, we can write any $x \in \mathcal{P}$ as $B x_\mathcal{B} + N x_\mathcal{N} = b$, so $x_\mathcal{B} = B^{-1}b - B^{-1}N x_\mathcal{N}$. Substituting into the objective:

$$c^\top x = c_\mathcal{B}^\top x_\mathcal{B} + c_\mathcal{N}^\top x_\mathcal{N} = \underbrace{c_\mathcal{B}^\top B^{-1}b}_{z_0} + \underbrace{\bigl(c_\mathcal{N}^\top - c_\mathcal{B}^\top B^{-1}N\bigr)}_{\bar{c}_\mathcal{N}^\top} x_\mathcal{N}.$$

The vector $\bar{c}_j = c_j - c_\mathcal{B}^\top B^{-1} A_j$ for $j \in \mathcal{N}$ is the reduced cost. Since $x_\mathcal{N} \geq 0$, the objective can only decrease if some $\bar{c}_j < 0$. This gives:

Theorem 3.1 — Optimality Criterion

A BFS with basis $\mathcal{B}$ is optimal if and only if $\bar{c}_j \geq 0$ for all $j \in \mathcal{N}$.

If $\bar{c}_e < 0$ for some $e$ and the direction $d = B^{-1}A_e \leq 0$ component-wise, then the LP is unbounded.

This section is the hinge for several later topics: reduced costs become the dual slacks in §6, the zero reduced-cost condition becomes complementary slackness in §8, and the same formulas drive objective sensitivity in §10.

Note that $\bar{c}_i = 0$ for all $i \in \mathcal{B}$ (by construction). The dual variables, or shadow prices, are $y^* = B^{-\top}c_\mathcal{B}$, and the reduced costs rewrite as $\bar{c}_j = c_j - (y^*)^\top A_j$.

Optimality Certificate View

At an optimal basis, the dual vector is not just an internal computation. It is an independently checkable certificate of optimality.

$$y^*=B^{-\top}c_B,\qquad A^\top y^*\le c,\qquad b^\top y^*=c^\top x^*.$$
  1. Dual feasibility is verified by checking $A^\top y^*\le c$, equivalently $\bar c_j=c_j-A_j^\top y^*\ge 0$ for every column.
  2. Value matching is verified by checking $b^\top y^*=c^\top x^*$.
  3. Once both conditions hold, no re-run of simplex is needed: weak duality certifies that $x^*$ and $y^*$ are optimal.

This is one of the most important practical meanings of reduced costs: they turn an optimal basis into a short, verifiable certificate.

Scaling and Reduced Costs

Column scaling changes the numerical size of reduced costs, which is why practical solvers scale models before simplex iterations.

$$A_j \mapsto \hat A_j=\alpha_j A_j,\qquad x_j \mapsto \hat x_j = x_j/\alpha_j,\qquad \alpha_j>0.$$
  1. The feasible set is represented in a numerically different coordinate system, even though the underlying LP is equivalent.
  2. Under this change, reduced costs are computed relative to the scaled columns, so their magnitudes change with the scaling factors.
  3. Good scaling reduces large coefficient disparities, making pricing, ratio tests, and basis solves numerically more reliable.
  4. For that reason, production solvers often use scaled reduced costs for pivot selection rather than trusting raw coefficient magnitudes.

Block-Matrix View at a Basis

At basis $\mathcal{B}$, write columns and costs in block form:

$$A=[B\;N],\qquad x=\begin{bmatrix}x_\mathcal{B}\\x_\mathcal{N}\end{bmatrix},\qquad c=\begin{bmatrix}c_\mathcal{B}\\c_\mathcal{N}\end{bmatrix}.$$
  1. Constraint decomposition: $Bx_\mathcal{B}+Nx_\mathcal{N}=b$.
  2. Isolate basic block: $x_\mathcal{B}=B^{-1}b-B^{-1}Nx_\mathcal{N}$.
  3. Substitute into objective: $$c^\top x =c_\mathcal{B}^\top B^{-1}b +\big(c_\mathcal{N}^\top-c_\mathcal{B}^\top B^{-1}N\big)x_\mathcal{N}.$$
  4. Define shadow prices $y^\top:=c_\mathcal{B}^\top B^{-1}$ so: $$\bar c_\mathcal{N}^\top =c_\mathcal{N}^\top-y^\top N,\qquad \bar c_j=c_j-y^\top A_j.$$

This identity is the direct algebraic bridge between primal basis computations and dual feasibility checks.

Quick Check — Reduced Costs

Question: If $\bar c = (-2,0,1,-3)$ in a minimization problem, which variable enters under Dantzig's rule?

Answer: The variable with reduced cost $-3$ enters, because Dantzig's rule chooses the most negative reduced cost.

§04 The Pivot Operation

Select an entering variable $x_e$ with $\bar{c}_e < 0$. As $x_e$ increases from 0, basic variables change as $x_\mathcal{B} = \bar{b} - d\,\theta$ where $\bar{b} = B^{-1}b$ and $d = B^{-1}A_e$. Feasibility requires $\bar{b} - d\theta \geq 0$.

Definition 4.1 — Minimum Ratio Test

The step length and leaving variable index are:

$$\theta^* = \min_{i\,:\,d_i > 0}\frac{\bar{b}_i}{d_i} = \frac{\bar{b}_\ell}{d_\ell}.$$

Variable $x_\ell$ (with $\ell \in \mathcal{B}$) leaves the basis; $x_e$ enters. The objective improves by $\Delta z = -\theta^* \bar{c}_e \geq 0$.

Degeneracy Appears at the Ratio Test

The minimum ratio test can legitimately return $\theta^*=0$. When that happens, the basis changes but the current point does not.

$$\theta^*=0\quad\Longrightarrow\quad x_\mathcal{B}^{\text{new}}=\bar b-d\,\theta^*=\bar b.$$
  1. If some basic variable already has value $0$, the basis is degenerate.
  2. If that zero basic variable wins the ratio test, the pivot is algebraically valid but there is no movement in primal space.
  3. So the simplex method can stall: different bases can represent the same BFS and objective value.
  4. Without an anti-cycling rule, the algorithm can revisit the same bases forever; that is exactly how cycling occurs.

Tiny example: if two ratio-test ties both have value $0$, the method may pivot from basis $B_1$ to basis $B_2$ while keeping the same BFS, then later pivot back from $B_2$ to $B_1$ with the point still unchanged. Bland's rule blocks this by always taking the smallest eligible indices, so the repeated loop cannot continue indefinitely.

Theorem 4.2 — Bland's Anti-Cycling Rule

If we always choose both the entering and leaving variable as the eligible candidate with the smallest index, the simplex method terminates in finitely many steps without cycling.

  1. Assume for contradiction that the method cycles through a finite list of bases and eventually returns to one of them.
  2. Among all variables that enter or leave during the cycle, let $j$ be the smallest index.
  3. Since the basis repeats, variable $j$ must both enter at some pivot and leave at another pivot.
  4. When $j$ enters, Bland's rule says every other eligible entering variable has index larger than $j$.
  5. When $j$ later leaves, the pivot relations imply that some smaller-index eligible variable would have had to enter or leave earlier in the cycle to make this possible.
  6. That contradicts the choice of $j$ as the smallest index ever selected. Hence no cycle can occur.

The proof is short but important: Bland's rule is not a heuristic tie-breaker, it is a termination guarantee.

The new tableau is obtained via elementary row operations with pivot element $a_{\ell e} = d_\ell$:

Pivot Rules $$a'_{ij} = \begin{cases} a_{\ell j}\,/\,a_{\ell e} & i = \ell,\\[4pt] a_{ij} - \dfrac{a_{ie}}{a_{\ell e}}\,a_{\ell j} & i \neq \ell. \end{cases}$$
Why the Tableau Update Is Gaussian Elimination

The tableau is just the constraint system written as an augmented matrix. A pivot is the same elementary elimination step used to turn the entering column into a new basis column.

$$[A\mid b]\quad\longrightarrow\quad [A'\mid b'].$$
  1. Choose pivot column $e$ and pivot row $\ell$ so that the entering column will replace the leaving basic column.
  2. Normalize row $\ell$ by dividing by the pivot element $a_{\ell e}$; this makes the pivot entry equal to $1$.
  3. Subtract suitable multiples of the pivot row from every other row; this makes all other entries in column $e$ equal to $0$.
  4. After these row operations, column $e$ is a unit vector, so it can serve as a basis column exactly like in Gaussian elimination.
  5. The right-hand side undergoes the same row operations, which is why the updated tableau still represents an equivalent linear system.

So the tableau is not a separate object from linear algebra; it is Gaussian elimination organized to preserve basis structure after each pivot.

§05 The Simplex Algorithm & Phase I

Two-Phase Simplex: End-to-End Flow

Simplex is operationally a two-stage method. Phase I finds (or disproves) primal feasibility; Phase II optimizes the original objective from the feasible basis returned by Phase I.

Workflow — How Phase I Connects to Phase II
  1. START with original LP $(P):\min\{c^\top x:Ax=b,\ x\ge 0\}$.
  2. If a feasible BFS is already known, skip Phase I and go directly to Phase II.
  3. Otherwise solve Phase I auxiliary LP: $$\min_{x,a\ge 0}\mathbf 1^\top a\quad\text{s.t.}\quad Ax+a=b.$$
  4. IF Phase I optimum $z_1^*>0$ THEN RETURN original LP infeasible.
  5. IF $z_1^*=0$, remove artificial variables and clean the basis (pivot out any basic artificial at value $0$ when possible).
  6. Use the resulting feasible basis $\mathcal B^{(0)}$ for the original model and switch objective from $\mathbf 1^\top a$ to $c^\top x$.
  7. Run Phase II simplex iterations until optimality ($\bar c\ge 0$), unboundedness, or stopping tolerance.

Two-Phase Flow Diagram

Phase Transition Conditions $$z_1^*>0\Rightarrow \mathcal P=\emptyset,\qquad z_1^*=0\Rightarrow \exists\ x^{(0)}\ge 0,\ Ax^{(0)}=b\ \text{(Phase II start)}.$$
Algorithm 5.1 — Simplex Method (Phase II, Given a Feasible Basis)
  1. INPUT feasible basis $\mathcal{B}$ with $B=A_\mathcal{B}$ and $\bar b=B^{-1}b\ge 0$.
  2. Compute reduced costs $\bar c_j \leftarrow c_j-c_\mathcal{B}^\top B^{-1}A_j,\ \forall j\in\mathcal{N}$.
  3. IF $\bar c_j\ge 0,\ \forall j\in\mathcal{N}$ THEN RETURN optimal BFS.
  4. Select entering index $e \leftarrow \arg\min\{\bar c_j:\ j\in\mathcal{N}\}$ (or Bland).
  5. Set direction $d \leftarrow B^{-1}A_e$.
  6. IF $d\le 0$ THEN RETURN unbounded.
  7. Choose leaving row $\ell \leftarrow \arg\min\{\bar b_i/d_i:\ d_i>0\}$ and $\theta^\*=\bar b_\ell/d_\ell$.
  8. Pivot: $\mathcal{B}\leftarrow(\mathcal{B}\setminus\{\ell\})\cup\{e\}$; update tableau and $\bar b$.
  9. GOTO line 2.

Phase I — Finding an Initial BFS

When no obvious starting BFS is available, introduce artificial variables $a \in \mathbb{R}^m_{\geq 0}$ and solve:

(P₁) $$\min_{x,\,a \geq 0}\;\mathbf{1}^\top a \quad \text{s.t.}\quad Ax + a = b.$$

Build the initial Phase I basis explicitly:

  1. Use augmented matrix $\hat{A}=[A\;I_m]$ with variables $(x,a)$.
  2. Choose initial basis as the artificial columns: $B_0=I_m$.
  3. Set nonbasic original variables to zero: $x=0$.
  4. Solve $B_0a=b \Rightarrow a=b$; if $b\ge 0$, this is immediately feasible.
  5. Run simplex on (P₁). If optimum is $0$, remove artificials and keep a feasible basis for the original constraints as the Phase II warm start.
Big-M Method vs. Two-Phase Method

The other standard textbook approach is the Big-M method. Instead of solving a separate Phase I problem, it penalizes artificial variables directly in the original objective.

$$\min\ c^\top x + M\mathbf 1^\top a\qquad (M\gg 1).$$
  1. Big-M: conceptually simpler because it uses one optimization model, but the choice of $M$ is delicate and can create serious numerical scaling problems.
  2. Two-phase: slightly more bookkeeping, but cleaner computationally because feasibility and optimization are separated.
  3. For actual solvers, two-phase is usually preferred: it avoids contaminating the original objective with huge coefficients and is more numerically stable.
  4. For teaching, both are worth knowing, but two-phase gives the clearer interpretation of what the algorithm is doing.
Worked Example — From Phase I to Phase II

Consider the LP

$$\begin{aligned} \min \quad & -x_1-3x_2\\ \text{s.t.}\quad & x_1+x_2+x_3=4,\\ & 2x_1+x_2+x_3=5,\\ & x_1,x_2,x_3\ge 0. \end{aligned}$$

There is no obvious identity basis in the original columns, so we start with Phase I:

$$\begin{aligned} \min \quad & a_1+a_2\\ \text{s.t.}\quad & x_1+x_2+x_3+a_1=4,\\ & 2x_1+x_2+x_3+a_2=5,\\ & x,a\ge 0. \end{aligned}$$
  1. Initial Phase I basis: choose $(a_1,a_2)$, so $(x_1,x_2,x_3)=(0,0,0)$ and $(a_1,a_2)=(4,5)$. Hence the initial Phase I objective is $z_1=9$.
  2. What Phase I returns: after simplex pivots drive the artificial variables to zero, the final Phase I basis is read directly from the tableau. In this example, the final basic columns are $(x_1,x_3)$ and the artificial columns are nonbasic at value $0$.

  1. With basis $(x_1,x_3)$, we have $c_B=(-1,0)$ and therefore $$y^\top=c_B^\top B^{-1}=(-1,0)\begin{bmatrix}-1&1\\2&-1\end{bmatrix}=(1,-1).$$
  2. The only nonbasic variable is $x_2$, with column $A_2=(1,1)^\top$. Its reduced cost is $$\bar c_2=c_2-y^\top A_2=-3-(1,-1)\begin{bmatrix}1\\1\end{bmatrix}=-3.$$
  3. Since $\bar c_2<0$, $x_2$ enters. The pivot direction is $$d=B^{-1}A_2=\begin{bmatrix}-1&1\\2&-1\end{bmatrix}\begin{bmatrix}1\\1\end{bmatrix} =\begin{bmatrix}0\\1\end{bmatrix}.$$
  4. Current basic values are $\bar b=B^{-1}b=(1,3)^\top$, so the ratio test gives $$\theta^*=\min_{i:d_i>0}\frac{\bar b_i}{d_i}=\frac{3}{1}=3.$$ Thus $x_3$ leaves and $x_2$ enters.
  5. The new basis is $(x_1,x_2)$, giving $$x=(1,3,0),\qquad c^\top x=-1-9=-10.$$ Since there are no remaining nonbasic original variables with negative reduced cost, this basis is optimal.

This example shows the full logic: Phase I proves feasibility and produces a basis; Phase II reuses that basis and continues optimizing the true objective.

Worked Example — Degeneracy and Stalling

Degeneracy means a basic variable is already zero. In that case, a pivot may change the basis without changing the current point.

$$\begin{aligned} \min \quad & -x_1-x_2\\ \text{s.t.}\quad & x_1+s_1=0,\\ & x_2+s_2=2,\\ & x_1+x_2+s_3=2,\\ & x,s\ge 0. \end{aligned}$$

At the slack basis $(s_1,s_2,s_3)$, the basic values are

$$\bar b=\begin{bmatrix}0\\2\\2\end{bmatrix}.$$

So the basis is feasible, but degenerate because $s_1=0$ is basic.

  1. If $x_1$ has negative reduced cost, it is a valid entering variable.
  2. Its pivot column is $d=(1,0,1)^\top$.
  3. The ratio test gives $$\theta^*=\min\left\{\frac{0}{1},\frac{2}{1}\right\}=0.$$
  4. So a pivot occurs, but the step length is zero: the basis changes, while the point itself does not move.

This is why degeneracy can produce stalling. The simplex method is still valid, but several pivots may occur without objective improvement. That is exactly why anti-cycling rules such as Bland's rule, perturbation, or lexicographic tie-breaking are needed in practice.

Common Mistakes — Phase I and Phase II
  1. Misreading the Phase I objective: if $z_1^*>0$, the original LP is infeasible. Only $z_1^*=0$ allows a transition to Phase II.
  2. Forgetting basis cleanup: after Phase I, artificial variables must be removed from the basis whenever possible before optimizing the original objective.
  3. Restarting from scratch: Phase II should inherit the feasible basis produced by Phase I; that basis is the whole point of the auxiliary problem.
  4. Mixing sign conventions: the reduced-cost optimality test depends on whether the primal is written as a minimization or maximization problem.

Revised Simplex Method

The tableau form is pedagogically useful, but large-scale LP solvers typically use the revised simplex method. Instead of storing and updating the full tableau, it stores only basis information and computes needed quantities on demand.

Core Quantities $$\bar b = B^{-1}b,\qquad y^\top = c_\mathcal{B}^\top B^{-1},\qquad \bar c_j = c_j - y^\top A_j.$$
Worked Example — Revised Simplex as Linear Solves

Use the same Phase II starting basis from the previous example. The LP is

$$\begin{aligned} \min \quad & -x_1-3x_2\\ \text{s.t.}\quad & x_1+x_2+x_3=4,\\ & 2x_1+x_2+x_3=5,\\ & x_1,x_2,x_3\ge 0, \end{aligned} \qquad B=[A_1\ A_3]=\begin{bmatrix}1&1\\2&1\end{bmatrix}.$$

The current basis is $(x_1,x_3)$, so $c_B=(-1,0)$ and the nonbasic column is $A_2=(1,1)^\top$. In revised simplex, we do not form $B^{-1}$ explicitly. We solve small auxiliary systems instead.

  1. Basic values: solve $$B\bar b=b,\qquad \begin{bmatrix}1&1\\2&1\end{bmatrix}\bar b= \begin{bmatrix}4\\5\end{bmatrix}.$$ Subtracting the first row from the second gives $\bar b_1=1$, then $\bar b_2=3$. Hence $$\bar b=\begin{bmatrix}1\\3\end{bmatrix},$$ so the current BFS is $(x_1,x_2,x_3)=(1,0,3)$.
  2. Dual multipliers: solve the transposed system $$B^\top y=c_B,\qquad \begin{bmatrix}1&2\\1&1\end{bmatrix}y= \begin{bmatrix}-1\\0\end{bmatrix}.$$ From $y_1+y_2=0$ and $y_1+2y_2=-1$, we get $$y=\begin{bmatrix}1\\-1\end{bmatrix}.$$
  3. Reduced cost of the nonbasic variable: $$\bar c_2=c_2-A_2^\top y=-3-\begin{bmatrix}1&1\end{bmatrix}\begin{bmatrix}1\\-1\end{bmatrix}=-3.$$ Since $\bar c_2<0$, $x_2$ should enter the basis.
  4. Pivot direction: solve $$Bd=A_2,\qquad \begin{bmatrix}1&1\\2&1\end{bmatrix}d= \begin{bmatrix}1\\1\end{bmatrix}.$$ Subtracting the first equation from the second gives $d_1=0$, and then $d_2=1$. So $$d=\begin{bmatrix}0\\1\end{bmatrix}.$$
  5. Ratio test: only the second component of $d$ is positive, so $$\theta^*=\frac{\bar b_2}{d_2}=\frac{3}{1}=3.$$ Therefore $x_3$ leaves and $x_2$ enters.

The updated basis is $(x_1,x_2)$ and the new solution is $x=(1,3,0)$.

Implementation Principle $$\text{Revised simplex uses solves with }B\text{ and }B^\top,\ \text{not an explicit }B^{-1}.$$

That distinction matters computationally: industrial solvers maintain sparse factorizations of $B$ and repeatedly solve systems like $Bu=r$ and $B^\top v=s$ because it is cheaper, more stable, and more scalable than computing the inverse itself.

  1. Feasibility check: current BFS is feasible iff $\bar b\ge 0$.
  2. Pricing: compute reduced costs using $y^\top=c_\mathcal{B}^\top B^{-1}$; choose entering variable from $\bar c_j<0$.
  3. Direction: solve $Bd=A_e$ (equivalently $d=B^{-1}A_e$).
  4. Ratio test: $\theta^\*=\min_{i:d_i>0}\bar b_i/d_i$ and leaving row $\ell$ at the minimizer.
  5. Basis update: replace column $A_{B_\ell}$ with $A_e$ in $B$; update factorizations of $B$.

In practice, $B^{-1}$ is not formed explicitly. Solvers maintain sparse LU factors (or product-form/eta updates) and perform repeated triangular solves for $Bv=u$ and $B^\top w=r$.

Algorithm 5.3 — Revised Simplex (Primal)
  1. INPUT feasible basis $\mathcal B$, sparse factorization of $B=A_\mathcal B$.
  2. Solve $B^\top y=c_\mathcal B$ to obtain dual multipliers $y$.
  3. Compute reduced costs $\bar c_j=c_j-A_j^\top y$ for $j\in\mathcal N$.
  4. IF $\bar c_j\ge 0,\ \forall j\in\mathcal N$ THEN RETURN optimal basis.
  5. Select entering index $e$ with $\bar c_e<0$ (Dantzig, steepest-edge, or Bland).
  6. Solve $Bd=A_e$ for pivot direction $d$.
  7. IF $d\le 0$ THEN RETURN unbounded.
  8. Solve $B\bar b=b$ (or update $\bar b$ incrementally), then ratio test for leaving row $\ell$.
  9. Update basis $\mathcal B\leftarrow(\mathcal B\setminus\{B_\ell\})\cup\{e\}$ and refresh/update factorization.
  10. GOTO line 2.
Why Revised Simplex Scales Better

For sparse LPs with millions of variables, storing a full tableau is prohibitive. Revised simplex exploits sparsity and performs mostly sparse linear algebra kernels, making memory and runtime much more favorable in industrial settings (network flow, logistics, energy, MIP node re-optimization).

Pricing Rules in Practice

The entering-variable rule strongly affects iteration count. Modern revised simplex implementations usually prefer scaled pricing over plain Dantzig in difficult instances.

Scaled Pricing $$\text{Dantzig:}\quad e=\arg\min_{j\in\mathcal N}\bar c_j,$$ $$\text{Steepest-edge proxy:}\quad e=\arg\min_{j\in\mathcal N}\frac{\bar c_j}{\sqrt{w_j}},\qquad w_j\approx\|B^{-1}A_j\|_2^2.$$
  1. Dantzig pricing: cheapest reduced cost to evaluate, but may zig-zag under degeneracy.
  2. Steepest-edge pricing: normalizes by step geometry and often reduces pivots substantially.
  3. Devex: low-cost approximation of steepest-edge; widely used default in high-performance solvers.
Numerical Stability and Degeneracy Handling
  1. Refactorization policy: periodic fresh LU of $B$ to control fill-in and accumulated roundoff from updates.
  2. Feasibility tolerances: treat tiny violations with tolerances (e.g., $10^{-7}$ to $10^{-9}$ range depending on scaling).
  3. Perturbation/lexicographic tie-break: avoids stalling and cycling when many ratios are nearly identical.
  4. Bound flipping / dual re-optimization: in branch-and-bound nodes, dual simplex often re-solves faster than restarting primal.
  5. Scaling: row/column equilibration improves conditioning and pivot reliability before simplex iterations.

Notation Checkpoint — Before Duality

Before starting duality, keep the following objects straight. Almost every later formula is built from them:

$$A=[B\;N],\qquad x=\begin{bmatrix}x_B\\x_N\end{bmatrix},\qquad x_B=B^{-1}b,\qquad y^\top=c_B^\top B^{-1},\qquad \bar c_j=c_j-A_j^\top y.$$
  1. $B$: the current basis matrix, formed from the basic columns.
  2. $x_B$: the current basic variables, obtained from $Bx_B=b$.
  3. $y$: the dual multiplier or shadow-price vector associated with the basis.
  4. $\bar c_j$: the reduced cost of column $A_j$, which measures whether entering that variable can improve the objective.
§06 LP Duality & Weak Duality

Every LP (the primal) has an associated dual LP. For the standard-form primal (P), the dual is derived by attaching multipliers $y \in \mathbb{R}^m$ to the equality constraints:

(D) $$\begin{aligned} \max_{y \in \mathbb{R}^m} \quad & b^\top y \\ \text{s.t.} \quad & A^\top y \leq c. \end{aligned}$$

Dual Derivation in Matrix Form

Start from the primal standard form and derive the dual step by step using the Lagrangian:

  1. Primal: $$\min_{x\ge 0}\;c^\top x\quad\text{s.t.}\quad Ax=b.$$
  2. Lagrangian with equality multipliers $y\in\mathbb{R}^m$: $$\mathcal L(x,y)=c^\top x+y^\top(b-Ax)=b^\top y+x^\top(c-A^\top y).$$
  3. Dual function: $$g(y):=\inf_{x\ge 0}\mathcal L(x,y).$$
  4. Characterize finiteness: $$g(y)=\begin{cases} b^\top y, & A^\top y\le c,\\ -\infty, & \text{otherwise}. \end{cases}$$
  5. Maximize the dual function over $y$: $$\max_y g(y)\equiv \max_{y\in\mathbb{R}^m}\{b^\top y:\;A^\top y\le c\}.$$

Intuitively, the dual variables $y_i$ price each unit of resource $b_i$. The dual constraints $A^\top y \leq c$ say that the "shadow price" of each activity cannot exceed its direct cost.

Economic Interpretation — Shadow Prices as Marginal Values

The dual vector is not just algebra. At optimality, it gives the local marginal value of relaxing each resource constraint.

$$y_i^*=\frac{\partial v(b)}{\partial b_i}\quad\text{locally, as long as the optimal basis stays unchanged.}$$
  1. Let $v(b)$ be the optimal primal value as a function of the resource vector $b$.
  2. If the current basis $B$ remains optimal after a small RHS perturbation $\Delta b$, then $$x_B(\Delta b)=B^{-1}(b+\Delta b),\qquad v(b+\Delta b)=c_B^\top B^{-1}(b+\Delta b).$$
  3. Therefore $$v(b+\Delta b)-v(b)=y^{*\top}\Delta b,\qquad y^{*\top}=c_B^\top B^{-1}.$$
  4. So $y_i^*$ is the local marginal value of one additional unit of resource $i$.
  5. Equivalently, in this minimization setting, a small change $\Delta b_i$ changes the optimal value by approximately $y_i^*\Delta b_i$. That is the precise meaning of the shadow price.

This is the rigorous bridge from duality to sensitivity analysis in §10.

Column-Wise Interpretation of $A^\top y \le c$

Let $A=[A_1\;\cdots\;A_n]$ with columns $A_j\in\mathbb{R}^m$. Then:

$$A^\top y \le c \quad\Longleftrightarrow\quad A_j^\top y \le c_j,\;\; j=1,\ldots,n.$$

So each primal variable/column generates exactly one dual inequality. This is why reduced costs are dual slacks:

$$\bar c_j = c_j - A_j^\top y \ge 0.$$
Dual of the Dual Returns the Primal

The symmetry is exact. Starting from

$$\text{(D)}\qquad \max\{b^\top y:\ A^\top y\le c,\ y\in\mathbb R^m\},$$

rewrite the inequalities as $A^\top y+s=c$ with $s\ge 0$. The dual of this maximization problem is

$$\min\{c^\top x:\ Ax=b,\ x\ge 0\},$$

which is exactly the original primal. So primal and dual are not two unrelated constructions; they are mirror images of the same LP structure.

Theorem 6.1 — Weak Duality

For any primal-feasible $x$ and dual-feasible $y$:

$$b^\top y \;\leq\; c^\top x.$$

Equivalently, every dual objective value is a lower bound on every primal objective value.

Proof of Weak Duality

Use the Lagrangian of the primal problem (with multipliers $y$ for $Ax=b$):

$$\mathcal{L}(x,y)=c^\top x+y^\top(b-Ax)=b^\top y+x^\top(c-A^\top y).$$
  1. Define the dual function: $g(y):=\inf_{x\ge 0}\mathcal{L}(x,y)$.
  2. If $A^\top y\le c$, then $c-A^\top y\ge 0$, so for every $x\ge 0$: $$x^\top(c-A^\top y)\ge 0\quad\Rightarrow\quad \mathcal{L}(x,y)\ge b^\top y.$$
  3. Taking infimum over $x\ge 0$ gives $g(y)\ge b^\top y$; and with $x=0$, $\mathcal{L}(0,y)=b^\top y$, hence $g(y)=b^\top y$.
  4. For any primal-feasible $x$ ($Ax=b$), $\mathcal{L}(x,y)=c^\top x$.
  5. Since $g(y)=\inf_{u\ge 0}\mathcal{L}(u,y)\le \mathcal{L}(x,y)=c^\top x$, we obtain: $$b^\top y=g(y)\le c^\top x.$$

Therefore, for every primal-feasible $x$ and dual-feasible $y$, $b^\top y\le c^\top x$.

An immediate consequence: if $c^\top x = b^\top y$ for some feasible pair $(x, y)$, then both are simultaneously optimal — a powerful certificate of optimality requiring no further computation.

Corollary 6.2 — Duality Gap and Certificates

The duality gap is $c^\top x - b^\top y \geq 0$. If the gap equals zero, both $x$ and $y$ are optimal. Moreover:

• If (P) is unbounded below, then (D) is infeasible.
• If (D) is unbounded above, then (P) is infeasible.

Quick Check — Weak Duality

Question: If feasible $x$ and $y$ satisfy $c^\top x=b^\top y$, what can you conclude?

Answer: Both are optimal, because the duality gap is zero.

§07 Strong Duality

Weak duality gives a bound; strong duality asserts the bound is tight. The proof uses the Farkas Lemma — a theorem of alternatives that is the LP analogue of KKT conditions.

Lemma 7.1 — Farkas' Lemma

For $A \in \mathbb{R}^{m\times n}$, $b \in \mathbb{R}^m$, exactly one of the following systems has a solution:

(I) $\exists\, x \geq 0 : Ax = b$

(II) $\exists\, y \in \mathbb{R}^m : A^\top y \leq 0,\; b^\top y > 0$

Proof of Farkas' Lemma via the Auxiliary LP

This proof is algebraic and constructive: it uses the same Phase I machinery already developed for simplex, not the separating hyperplane theorem as a black box.

Step 1: (I) and (II) cannot hold simultaneously.

  1. Assume $x\ge 0,\ Ax=b$ and also $A^\top y\le 0,\ b^\top y>0$.
  2. Then $$b^\top y = y^\top Ax = x^\top A^\top y\le 0,$$ because $x\ge 0$ and $A^\top y\le 0$ componentwise.
  3. This contradicts $b^\top y>0$.

Step 2: if (I) fails, construct a certificate for (II).

  1. Consider the auxiliary Phase I LP $$\min_{x,a\ge 0}\ \mathbf 1^\top a\qquad \text{s.t.}\qquad Ax+a=b.$$ After the usual row preprocessing to ensure $b\ge 0$, this problem has an obvious starting basis in the artificial variables.
  2. If the optimum were $0$, then necessarily $a^*=0$, so $Ax^*=b$ with $x^*\ge 0$, which is system (I). Therefore, if (I) is infeasible, the auxiliary optimum must satisfy $\eta^*>0$.
  3. Let $\hat A=[A\ I]$ and let $B$ be an optimal basis of the auxiliary LP. Its cost vector is $\hat c=(0,\mathbf 1)$, so the simplex shadow-price vector is $$y^*=B^{-\top}\hat c_B.$$
  4. Optimality of the basis means all reduced costs are nonnegative: $$\hat c_j-\hat A_j^\top y^*\ge 0\qquad \text{for every column } \hat A_j.$$ For the original columns $A_j$ this becomes $$0-A_j^\top y^*\ge 0\qquad\Longrightarrow\qquad A^\top y^*\le 0.$$
  5. The auxiliary objective value equals the dual value read from the final basis: $$b^\top y^*=b^\top B^{-\top}\hat c_B=(B^{-1}b)^\top \hat c_B=\hat c^\top \hat x^*=\eta^*>0.$$ Thus $y^*$ satisfies $A^\top y^*\le 0$ and $b^\top y^*>0$, which is system (II).

Therefore exactly one of (I) or (II) is feasible.

Farkas Lemma (Max-Form Convention)

For $A\in\mathbb{R}^{m\times n}$ and $b\in\mathbb{R}^m$, exactly one holds:

(1) $\exists\,x\in\mathbb{R}_{\ge 0}^n$ such that $Ax=b$.

(2) $\exists\,y\in\mathbb{R}^m$ such that $A^\top y\ge 0$ and $b^\top y<0$.

If both held, then $$b^\top y=(Ax)^\top y=x^\top A^\top y\ge 0,$$ contradicting $b^\top y<0$.

Strong Duality via Farkas (Max-Form Derivation)

Let $$p^*=\max\{c^\top x:\ Ax=b,\ x\ge 0\},\qquad c^\top x^*=p^*.$$

To obtain a dual optimizer, consider the system $$b^\top y\le p^*,\qquad A^\top y\ge c,$$ equivalently $$\begin{bmatrix} b^\top\\ -A^\top\end{bmatrix}y\le\begin{bmatrix} p^*\\ -c\end{bmatrix}.$$

Assume this system is infeasible. By Farkas (inequality form), there exists $\lambda\ge 0$ with $$\lambda^\top\begin{bmatrix} b^\top\\ -A^\top\end{bmatrix}=0,\qquad \lambda^\top\begin{bmatrix} p^*\\ -c\end{bmatrix}<0.$$

Write $\lambda=(\lambda_0,u)$ with $\lambda_0\ge 0,\ u\ge 0$. Then $$Au=\lambda_0 b,\qquad \lambda_0 p^*-c^\top u<0.$$

Case 1: $\lambda_0=0$. Then $Au=0$, $u\ge 0$, and $c^\top u>0$. Hence $x^*+tu$ remains feasible for $t\ge 0$ and improves the objective, contradicting optimality of $x^*$.

Case 2: $\lambda_0>0$. Set $x=u/\lambda_0$. Then $Ax=b$ and $$c^\top x=\frac{c^\top u}{\lambda_0}>p^*,$$ contradiction.

So the system is feasible: $\exists\,y$ with $A^\top y\ge c$ and $b^\top y\le p^*$. By weak duality, $b^\top y\ge c^\top x^*=p^*$, hence $b^\top y=p^*$ and strong duality follows.

Theorem 7.2 — Strong Duality

If (P) has an optimal solution $x^*$, then (D) also has an optimal solution $y^*$, and:

$$c^\top x^* = b^\top y^*.$$

Equivalently, when both (P) and (D) are feasible, the duality gap at optimality is exactly zero.

Proof of Strong Duality (via Simplex optimality)

Assumption. Let $x^*$ be an optimal primal BFS with basis $\mathcal B$, and let $B:=A_\mathcal B$ (invertible), $N:=A_\mathcal N$.

Step 1: write the primal basic structure explicitly.

$$x^*=\begin{bmatrix}x_\mathcal B^*\\x_\mathcal N^*\end{bmatrix},\qquad x_\mathcal N^*=0,\qquad Bx_\mathcal B^*=b,\qquad x_\mathcal B^*=B^{-1}b.$$

Step 2: reduced costs at an optimal basis.

$$\bar c_j:=c_j-c_\mathcal B^\top B^{-1}A_j\ge 0,\qquad j=1,\ldots,n.$$

Step 3: construct a dual candidate. Define

$$y^*:=B^{-\top}c_\mathcal B.$$

Step 4: prove dual feasibility line by line. For each column $A_j$:

$$\begin{aligned} c_j-A_j^\top y^* &= c_j-A_j^\top(B^{-\top}c_\mathcal B)\\ &= c_j-c_\mathcal B^\top B^{-1}A_j\\ &= \bar c_j\\ &\ge 0. \end{aligned}$$

Hence $A^\top y^*\le c$, so $y^*$ is dual feasible.

Step 5: compute the dual objective, equality by equality.

$$\begin{aligned} b^\top y^* &= b^\top B^{-\top}c_\mathcal B\\ &= (B^{-1}b)^\top c_\mathcal B\\ &= (x_\mathcal B^*)^\top c_\mathcal B\\ &= (x_\mathcal B^*)^\top c_\mathcal B + (x_\mathcal N^*)^\top c_\mathcal N\\ &= (x^*)^\top c\\ &= c^\top x^*. \end{aligned}$$

Step 6: conclude optimality via weak duality.

  1. Weak duality gives $b^\top y \le c^\top x$ for any primal-feasible $x$ and dual-feasible $y$.
  2. Apply it to $x^*$ (primal feasible) and $y^*$ (dual feasible): $b^\top y^* \le c^\top x^*$.
  3. From Step 5, $b^\top y^*=c^\top x^*$, so the inequality is tight.
  4. Therefore $x^*$ is primal optimal, $y^*$ is dual optimal, and the duality gap is zero.

This proof is remarkably constructive: the optimal dual solution is simply the shadow price vector $y^* = B^{-\top}c_\mathcal{B}$ read off from the final Simplex tableau. No separate dual computation is needed.

Concrete Example — Both Primal and Dual Infeasible

A simultaneous infeasibility example is easy to build in standard form:

$$\begin{aligned} \text{(P)}\qquad \min\quad & -x_1-x_2\\ \text{s.t.}\quad & x_1-x_2=1,\\ & x_1-x_2=0,\\ & x\ge 0. \end{aligned}$$

The two equality constraints are inconsistent, so the primal is infeasible.

Its dual is

$$\begin{aligned} \text{(D)}\qquad \max\quad & y_1\\ \text{s.t.}\quad & y_1+y_2\le -1,\\ & -y_1-y_2\le -1. \end{aligned}$$

These dual inequalities require simultaneously $$y_1+y_2\le -1 \qquad\text{and}\qquad y_1+y_2\ge 1,$$ which is impossible. So the dual is also infeasible.

§08 Complementary Slackness

Strong duality's zero-gap condition imposes tight structural conditions on the primal-dual solution pair. These conditions — complementary slackness — are the LP analogue of the KKT conditions familiar from nonlinear optimization.

Theorem 8.1 — Complementary Slackness (CS)

Let $x$ be primal-feasible and $y$ dual-feasible. Then $x$ and $y$ are simultaneously optimal if and only if both conditions hold:

$$\text{(CS-P):}\quad x_j\,(c_j - A_j^\top y) = 0 \quad \forall j \in \{1,\ldots,n\},$$ $$\text{(CS-D):}\quad (b_i - A_i^\top x)\,y_i = 0 \quad \forall i \in \{1,\ldots,m\}.$$

Since (P) is in standard form ($Ax=b$), (CS-D) is automatically satisfied. So optimality reduces to (CS-P): for each $j$, either $x_j = 0$ or $\bar{c}_j = c_j - A_j^\top y = 0$.

General Complementary Slackness Cheat Sheet

In practice, LP models are not always in standard form. The full KKT-style version of complementary slackness is what practitioners actually check on a candidate primal-dual pair.

Geometry Insight — Supporting Contour and Zero Duality Gap

At optimality, the dual vector defines a supporting hyperplane to the feasible region. Complementary slackness says the active primal variables lie exactly where that supporting contour touches the feasible set.

  1. Supporting contour: the optimal objective level set just touches the feasible region, rather than cutting through it.
  2. Active variables: when $x_j>0$, the corresponding reduced cost must vanish, so that direction is exactly tangent to the optimal contour.
  3. Inactive variables: when $x_j=0$, the reduced cost may stay positive; moving into that direction would worsen the objective.
Proof of Complementary Slackness

(⇒) Optimality implies CS.

  1. From strong duality at optimality: $c^\top x=b^\top y$.
  2. Expand the gap: $$0=c^\top x-b^\top y=c^\top x-(Ax)^\top y=\sum_{j=1}^n x_j(c_j-A_j^\top y)=\sum_j x_j\bar c_j.$$
  3. Each term is nonnegative ($x_j\ge 0,\;\bar c_j\ge 0$), so a zero sum implies $x_j\bar c_j=0$ for every $j$.

(⇐) CS implies optimality.

  1. Assume $x_j(c_j-A_j^\top y)=0$ for all $j$.
  2. Then $c^\top x=\sum_j x_j c_j=\sum_j x_j A_j^\top y=(Ax)^\top y=b^\top y$.
  3. By weak duality, equality of feasible primal/dual values implies both are optimal.

CS conditions have direct algorithmic significance: at the Simplex optimum, every nonbasic variable $x_j = 0$ satisfies $\bar{c}_j \geq 0$ (and typically $> 0$), while every basic variable has $\bar{c}_j = 0$ by construction. This is precisely complementary slackness.

This is also why sensitivity analysis works: once you know which reduced costs are zero and which are positive, you know which constraints or variables can change a little without changing the active optimal structure in §10.

Worked Example — Recover the Dual Optimum from a Known Primal Optimum

Complementary slackness is operational: once a primal optimum is known, it often determines the dual optimum immediately, without running simplex on the dual.

$$\begin{aligned} \text{(P)}\qquad \min\quad & -x_1-3x_2\\ \text{s.t.}\quad & x_1+x_2+x_3=4,\\ & 2x_1+x_2+x_3=5,\\ & x\ge 0. \end{aligned}$$

Suppose we already know the primal optimum is $$x^*=(1,3,0).$$ The dual is $$\max\ 4y_1+5y_2\qquad \text{s.t.}\qquad \begin{cases} y_1+2y_2\le -1,\\ y_1+y_2\le -3,\\ y_1+y_2\le 0. \end{cases}$$

  1. Because $x_1^*>0$, complementary slackness forces the first reduced cost to vanish: $$c_1-A_1^\top y=-1-(y_1+2y_2)=0.$$
  2. Because $x_2^*>0$, the second reduced cost must also vanish: $$c_2-A_2^\top y=-3-(y_1+y_2)=0.$$
  3. Solve the two equalities: $$y_1+2y_2=-1,\qquad y_1+y_2=-3 \quad\Longrightarrow\quad y^*=(-5,2).$$
  4. Check the variable with $x_3^*=0$ only for sign consistency: $$c_3-A_3^\top y^*=0-(-3)=3\ge 0,$$ so the third dual inequality is satisfied.
  5. Finally compare objective values: $$b^\top y^*=4(-5)+5(2)=-10=c^\top x^*.$$ Hence $y^*$ is the optimal dual solution.

This is the practical use of complementary slackness: positive primal variables identify the active dual constraints, and zero primal variables only need inequality checks.

Connection to KKT Conditions

The CS conditions are exactly the KKT conditions for (P). Forming the Lagrangian $\mathcal{L}(x,y,\mu) = c^\top x - y^\top(Ax - b) - \mu^\top x$, stationarity gives $c - A^\top y - \mu = 0$, so $\mu = c - A^\top y = \bar{c}$. The KKT conditions are:

$$\underbrace{Ax = b}_{\text{primal feas.}}, \quad \underbrace{\bar{c} \geq 0}_{\text{dual feas.}}, \quad \underbrace{\mu \geq 0}_{\bar{c} \geq 0}, \quad \underbrace{x_j\,\mu_j = 0\;\forall j}_{\text{CS-P}}.$$

For convex problems (and LP is convex), KKT conditions are both necessary and sufficient — fully consistent with our LP duality theory.

Answer: It must be zero, because complementary slackness requires $x_j\bar c_j=0$.

§09 The Dual Simplex Method

The Dual Simplex method maintains dual feasibility ($\bar{c} \geq 0$) throughout, gradually restoring primal feasibility ($\bar{b} \geq 0$). It is the algorithm of choice for:

• Sensitivity analysis — after modifying $b$ or adding constraints, primal feasibility is lost but dual feasibility is retained.
• Branch-and-bound — tightening bounds destroys primal feasibility; the dual Simplex re-optimizes cheaply.
• Cutting planes — adding a Gomory or other valid cut typically invalidates the current primal basic solution while leaving the dual information largely intact, so dual simplex is the natural re-optimization method.

Basis Status Map — Which Method Applies?
Primal Feasible? Dual Feasible? Status Typical Action
Yes Yes Optimal basis Stop
Yes No Feasible but not optimal Primal simplex is natural
No Yes Optimality conditions nearly preserved Dual simplex is natural
No No No useful optimality certificate yet Phase I, crash, or other repair strategy

This table is one of the most useful mental models in the whole class: primal simplex fixes dual infeasibility, dual simplex fixes primal infeasibility, and if both are already satisfied the basis is optimal.

Algorithm 9.1 — Dual Simplex
  1. INPUT dual-feasible basis with $\bar c\ge 0$ (allow $\bar b\not\ge 0$).
  2. IF $\bar b\ge 0$ THEN RETURN optimal BFS.
  3. Pick leaving row $\ell \leftarrow \arg\min_i \bar b_i$ (most negative RHS).
  4. Read pivot row coefficients $\bar a_{\ell j}\leftarrow (B^{-1}A)_{\ell j}$ for $j\in\mathcal N$.
  5. IF $\bar a_{\ell j}\ge 0,\ \forall j\in\mathcal N$ THEN RETURN primal infeasible.
  6. Pick entering index by dual ratio test: $$e\leftarrow \arg\min_{j:\bar a_{\ell j}<0}\frac{\bar c_j}{|\bar a_{\ell j}|}.$$
  7. Pivot and update basis $\mathcal B$; recompute tableau quantities.
  8. GOTO line 2.

The dual ratio test in Step 5 ensures that $\bar{c} \geq 0$ is maintained after each pivot — the dual of the primal ratio test that maintains $\bar{b} \geq 0$. Strong duality guarantees both procedures converge to the same optimal solution.

Why Step 5 Certifies Primal Infeasibility

If the chosen pivot row satisfies $\bar a_{\ell j}\ge 0$ for every nonbasic index $j$, the current negative basic variable cannot be repaired by any dual-simplex pivot.

  1. The violated basic variable is $x_{B_\ell}=\bar b_\ell<0$.
  2. When a nonbasic variable $x_j$ changes from $0$ to $\theta\ge 0$, the basic variables move as $$x_B^{\text{new}}=\bar b-\bar A_{\cdot j}\theta,$$ so row $\ell$ becomes $$x_{B_\ell}^{\text{new}}=\bar b_\ell-\bar a_{\ell j}\theta.$$
  3. If $\bar a_{\ell j}\ge 0$, then increasing $x_j$ cannot increase $x_{B_\ell}$; it either leaves it unchanged or makes it even smaller.
  4. Therefore no admissible nonbasic move can bring the violated constraint $x_{B_\ell}\ge 0$ back to feasibility.
  5. So the current dictionary contains an explicit certificate that the primal system has no feasible completion from this dual-feasible basis. In the language of alternatives, this is exactly the situation Farkas' lemma detects: the primal is infeasible.
Worked Example — Dual Simplex Re-Optimization

Start from the LP

$$\begin{aligned} \min \quad & -x_1-2x_2\\ \text{s.t.}\quad & x_1+x_2+s_1=4,\\ & x_1+2x_2+s_2=6,\\ & x_1,x_2,s_1,s_2\ge 0. \end{aligned}$$

Suppose we already solved it and have the optimal basis $(x_1,x_2)$. Its basis matrix and inverse are

$$B=\begin{bmatrix}1&1\\1&2\end{bmatrix},\qquad B^{-1}=\begin{bmatrix}2&-1\\-1&1\end{bmatrix}.$$

For the original right-hand side $b=(4,6)^\top$, the basic solution is

$$x_B=B^{-1}b=\begin{bmatrix}2&-1\\-1&1\end{bmatrix}\begin{bmatrix}4\\6\end{bmatrix} =\begin{bmatrix}2\\2\end{bmatrix},$$

so $(x_1,x_2,s_1,s_2)=(2,2,0,0)$.

The reduced costs of the nonbasic slacks are nonnegative: $$c_B=(-1,-2),\qquad y^\top=c_B^\top B^{-1}=(0,-1),$$ $$\bar c_{s_1}=0-y^\top\!\begin{bmatrix}1\\0\end{bmatrix}=0,\qquad \bar c_{s_2}=0-y^\top\!\begin{bmatrix}0\\1\end{bmatrix}=1.$$ Thus the basis is optimal.

Now change only the RHS to $$b'=\begin{bmatrix}1\\6\end{bmatrix}.$$ Reusing the same basis is natural in re-optimization. The new basic values become $$\bar b'=B^{-1}b'=\begin{bmatrix}2&-1\\-1&1\end{bmatrix}\begin{bmatrix}1\\6\end{bmatrix} =\begin{bmatrix}-4\\5\end{bmatrix}.$$

So primal feasibility is lost because the first basic variable is negative, but the reduced costs are unchanged and still satisfy $\bar c\ge 0$. This is exactly the dual-simplex setting.

  1. Leaving row: choose the most negative basic value, namely row $1$ with $\bar b'_1=-4$.
  2. Pivot row coefficients: for the nonbasic columns $(s_1,s_2)$, $$B^{-1}[A_{s_1}\ A_{s_2}] =\begin{bmatrix}2&-1\\-1&1\end{bmatrix}.$$ Hence the first pivot row is $(2,-1)$.
  3. Eligible entering variables: only columns with negative pivot-row coefficient are allowed, so only $s_2$ is eligible.
  4. Dual ratio test: $$\frac{\bar c_{s_2}}{|\bar a_{1,s_2}|}=\frac{1}{1}=1,$$ so $s_2$ enters and $x_1$ leaves.
  5. New basis: $(s_2,x_2)$ with $$B_{\text{new}}=\begin{bmatrix}0&1\\1&2\end{bmatrix},\qquad B_{\text{new}}^{-1}=\begin{bmatrix}-2&1\\1&0\end{bmatrix}.$$
  6. Updated basic values: $$x_B^{\text{new}}=B_{\text{new}}^{-1}b' =\begin{bmatrix}-2&1\\1&0\end{bmatrix}\begin{bmatrix}1\\6\end{bmatrix} =\begin{bmatrix}4\\1\end{bmatrix}\ge 0.$$ Therefore the new primal solution is feasible: $$(x_1,x_2,s_1,s_2)=(0,1,0,4).$$

Because dual simplex preserves $\bar c\ge 0$, primal feasibility plus dual feasibility implies optimality after the pivot. The new objective value is $$c^\top x=-2.$$

This is the practical use case for dual simplex: after a small model change, we keep the old basis, repair feasibility, and reach the new optimum without restarting from Phase I.

Common Mistakes — Dual Simplex
  1. Using dual simplex without dual feasibility: dual simplex starts from $\bar c\ge 0$ and repairs $\bar b$; if reduced costs are already wrong, this is not the right method.
  2. Choosing the wrong entering variable rule: in dual simplex, eligibility comes from negative pivot-row coefficients, not from negative reduced costs.
  3. Confusing the two simplex variants: primal simplex keeps $\bar b\ge 0$ and improves $\bar c$; dual simplex keeps $\bar c\ge 0$ and repairs $\bar b$.
  4. Missing the re-optimization use case: dual simplex is especially valuable after RHS, bound, or cut changes, where the old basis remains informative.
Quick Check — Dual Simplex

Question: Which feasibility condition is preserved during dual simplex iterations?

Answer: Dual feasibility. The reduced costs stay valid while primal feasibility is repaired.

§10 Sensitivity Analysis

Sensitivity analysis asks a practical question: once we have an optimal basis, how much can the data change before that basis stops being optimal? The answer is expressed through reduced costs, basic values, and the shadow prices carried by the dual multipliers.

What Stays Fixed in Local Sensitivity Analysis

Fix an optimal basis $\mathcal B$. Local sensitivity analysis studies perturbations for which the same basis remains optimal. Under that assumption:

$$x_B = B^{-1}b,\qquad y^\top = c_B^\top B^{-1},\qquad \bar c_N^\top = c_N^\top - c_B^\top B^{-1}N.$$

The basis remains optimal exactly while primal feasibility ($x_B\ge 0$) and dual feasibility ($\bar c_N\ge 0$ for a minimization problem) are preserved.

That statement is just the operational form of the earlier theory: primal feasibility comes from the basis geometry in §2, while dual feasibility and reduced costs come from §3 and complementary slackness in §8.

Local Sensitivity vs. Parametric Analysis

These are not the same task.

  1. Local sensitivity: vary one or a few parameters while insisting that the current basis remains optimal. This produces an interval or region of validity.
  2. Parametric analysis: continue changing the data even after leaving that region. Once a boundary is hit, the current basis stops being optimal and a pivot sequence produces a new basis and a new linear formula.
  3. So the value function is typically piecewise linear, not globally described by a single shadow-price formula.
  4. The local formulas in this section are exactly the first piece of that parametric picture.

Right-Hand Side Sensitivity

If only the right-hand side changes from $b$ to $b+\Delta b$, the basis matrix $B$ and the reduced costs remain unchanged. Only the basic values move:

$$x_B(\Delta b)=B^{-1}(b+\Delta b)=B^{-1}b+B^{-1}\Delta b.$$
  1. If $x_B(\Delta b)\ge 0$, the current basis stays primal feasible.
  2. Because reduced costs do not change, dual feasibility is preserved automatically.
  3. Hence the same basis remains optimal as long as the perturbed basic values stay nonnegative.
Worked Example — RHS Range with a Fixed Basis

Reuse the optimal basis from the dual-simplex example: $$B=\begin{bmatrix}1&1\\1&2\end{bmatrix},\qquad B^{-1}=\begin{bmatrix}2&-1\\-1&1\end{bmatrix},\qquad b=\begin{bmatrix}4\\6\end{bmatrix}.$$

The basic variables are $$x_B=B^{-1}b=\begin{bmatrix}2\\2\end{bmatrix}.$$

Now perturb only the first RHS entry: $b_1 \mapsto 4+\delta$. Then $$x_B(\delta)= \begin{bmatrix}2&-1\\-1&1\end{bmatrix} \begin{bmatrix}4+\delta\\6\end{bmatrix} = \begin{bmatrix}2+2\delta\\2-\delta\end{bmatrix}.$$

For the basis to stay feasible, both components must remain nonnegative: $$2+2\delta\ge 0,\qquad 2-\delta\ge 0.$$ Therefore $$-1\le \delta \le 2.$$

Inside this interval, the same basis stays optimal. If $\delta$ moves outside it, at least one basic variable becomes negative and re-optimization is needed, typically by dual simplex.

Objective-Coefficient Sensitivity

If the objective coefficients change while the basis is fixed, primal feasibility is unchanged, but dual feasibility may fail. For a nonbasic variable $x_j$, the reduced cost is

$$\bar c_j = c_j - c_B^\top B^{-1}A_j.$$
  1. Changing a nonbasic coefficient $c_j$ shifts only that reduced cost directly.
  2. Changing a basic coefficient changes the dual vector $y^\top=c_B^\top B^{-1}$ and therefore changes all reduced costs.
  3. For a minimization problem, the current basis stays optimal while all reduced costs remain nonnegative.
Shadow Prices and Marginal Values

When the basis stays optimal under a small RHS perturbation $\Delta b$, the change in the optimal objective is linear:

$$z^*(b+\Delta b)=z^*(b)+y^{*\top}\Delta b.$$

The components of $y^*$ are the shadow prices: they measure the local marginal value of one additional unit of each resource.

This formula is only local. It is valid exactly on the RHS range derived from $$B^{-1}(b+\Delta b)\ge 0.$$ Once that condition fails, the current basis is no longer feasible, the shadow-price formula stops being exact, and a new basis must be found.

Adding a New Variable (Column Generation View)

If a new activity with column $a_{\text{new}}$ and cost $c_{\text{new}}$ is added, the current basis stays optimal exactly when the new reduced cost is not improving.

$$\bar c_{\text{new}}=c_{\text{new}}-y^{*\top}a_{\text{new}}.$$
  1. For a minimization problem, if $\bar c_{\text{new}}\ge 0$, the new variable does not improve the objective and the current basis remains optimal.
  2. If $\bar c_{\text{new}}<0$, the new column violates dual feasibility and should enter the basis under primal simplex or column-generation logic.
  3. This is the simplest way to test whether a proposed new activity changes the solution: compute one reduced cost.

Adding a New Constraint

A new valid inequality or equality often keeps the old reduced costs intact but can make the old basic solution infeasible. That is exactly the regime where dual simplex is natural.

  1. Take the current optimal basis and append the new row.
  2. The old primal solution may violate the new constraint, so primal feasibility is lost.
  3. But the old dual multipliers and reduced-cost signs are often still usable, so dual feasibility is preserved or nearly preserved.
  4. Dual simplex then repairs the violated row without discarding the information in the current basis.

This is the basic sensitivity-analysis version of what happens in cutting-plane and branch-and-cut solvers.

Simultaneous Changes and the 100% Rule

Single-parameter ranges can be too conservative when several data values move at the same time. The classical 100% rule gives a safe joint condition.

100% Rule for Simultaneous RHS Changes

Suppose each RHS component $b_i$ has an allowable increase/decrease range while the basis stays feasible. For simultaneous changes $\Delta b_i$, define the fractional usage of each individual limit and sum them.

$$\sum_i \frac{|\Delta b_i|}{\text{allowable change for } b_i}\le 1 \quad\Longrightarrow\quad \text{the current basis remains feasible.}$$
  1. The same idea applies to objective-coefficient changes using reduced-cost ranges and dual feasibility instead of primal feasibility.
  2. The rule is sufficient, not necessary: if the sum exceeds $1$, the basis may still remain optimal, but the simple certificate is gone.
  3. Its value is practical: it converts many separate one-at-a-time reports into one safe simultaneous-change test.

Why Dual Simplex Appears Naturally

Sensitivity analysis and dual simplex are tightly linked. If an RHS or bound change destroys primal feasibility but leaves reduced costs valid, then dual simplex is exactly the right repair mechanism.

  1. RHS changes perturb $x_B=B^{-1}b$ directly.
  2. If some basic value becomes negative, the current basis is no longer primal feasible.
  3. If reduced costs are still nonnegative, the basis is dual feasible and dual simplex can re-optimize cheaply.
Common Mistakes — Sensitivity Analysis
  1. Thinking sensitivity is global: the formulas describe a local range where the same basis remains optimal, not arbitrary large perturbations.
  2. Mixing feasibility and optimality: RHS changes usually threaten primal feasibility; objective changes usually threaten dual feasibility.
  3. Misreading shadow prices: a shadow price is a local marginal value, not a guarantee of linear behavior after large changes.
Quick Check — Sensitivity

Question: If only the RHS changes and the same basis remains feasible, do the reduced costs change?

Answer: No. With the basis fixed, RHS changes alter the basic values but do not change the reduced costs.

§11 Complexity & Interior-Point Methods

The number of BFS is bounded by $\binom{n}{m}$, giving a worst-case exponential bound. Klee and Minty (1972) showed that, with Dantzig's pivot rule, the Simplex method visits all $2^n - 1$ vertices of a specially constructed $n$-dimensional cube. Yet in practice, the algorithm takes $O(m)$ to $O(m\log m)$ pivots.

Theorem 11.1 — Ellipsoid Method (Khachiyan, 1979)

The ellipsoid method was the first algorithm to prove that linear programming is solvable in polynomial time. Its importance is primarily theoretical: it established polynomial-time solvability, even though it is not competitive with simplex or modern interior-point methods in most practical LP instances.

Theorem 11.2 — Karmarkar's Algorithm

The interior-point projective method (Karmarkar, 1984) solves LP in polynomial time $O(n^{3.5}L)$, where $L$ is the input bit-length. Modern primal-dual interior-point methods achieve $O(\sqrt{n}\,L)$ iterations, each costing $O(n^3)$ per factorization.

Historical Complexity Narrative
  1. Simplex (1947): extremely effective in practice, but only exponential worst-case guarantees are known in general.
  2. Ellipsoid (1979): the first proof that LP is polynomial-time solvable.
  3. Karmarkar (1984): the first practically important polynomial-time interior-point breakthrough.
  4. Modern reality: neither ellipsoid nor interior-point methods displaced simplex across the board; simplex remains dominant in many sparse industrial LP and MIP workflows because of warm starts, re-optimization, and basis information.
Smoothed Analysis of Simplex

The modern explanation for simplex's practical speed is not worst-case analysis but smoothed analysis (Spielman and Teng, 2004).

  1. Worst-case instances like Klee-Minty are highly fragile and adversarial.
  2. If the input is subjected to tiny random perturbations, the expected running time of simplex becomes polynomial under the smoothed-analysis model.
  3. This helps explain the empirical fact that real LPs are rarely worst-case pathological, even though the theoretical worst case is exponential.

This is arguably the most important modern complexity result about simplex itself: it reconciles bad worst-case theory with excellent observed performance.

The question of whether LP admits a strongly polynomial algorithm (polynomial in $n$ and $m$, independent of $L$) remains one of the great open problems in theoretical computer science, listed by Smale in his "Mathematical Problems for the Next Century."

For practitioners: the Simplex method dominates in sparse, large-scale industrial LPs and MIPs (via dual Simplex in branch-and-bound). Interior-point methods excel at dense second-order cone and semidefinite programs — the natural territory of ML regularization, SVM dual, and covariance estimation.

Design Logic — Why These Headers Exist

A modern simplex implementation is really four coupled mathematical subsystems:

$$\text{Presolve} \;\longrightarrow\; \text{Basis Linear Algebra} \;\longrightarrow\; \text{Pricing / Pivot Control} \;\longrightarrow\; \text{Certificates + Postsolve}.$$
  1. presolver.h: transforms the model into an equivalent but easier LP, ideally with fewer rows/columns and better conditioning.
  2. simplex_lu.h: implements the numerical kernel for the repeated solves $$B d = A_e,\qquad B^\top y = c_B,\qquad Bx_B=b.$$
  3. simplex_aux.h: decides which entering candidate to trust when several are theoretically valid but numerically risky.
  4. simplex.h: orchestrates primal vs dual mode, tolerances, recovery, telemetry, and final certificates.

The theory says which equations must hold; the headers decide how to satisfy them reliably on finite-precision machines.

Algorithm 12.1 — Industrial Simplex Pipeline (Simplified)
  1. PRESOLVE remove fixed variables, tighten bounds, detect redundant constraints, aggregate singleton rows/columns.
  2. SCALE equilibrate rows/columns to improve conditioning and pivot reliability.
  3. CRASH construct a sparse near-feasible basis (logical/slack/network crash).
  4. CHOOSE MODE start primal or dual simplex based on infeasibility profile (often dual after bound/row changes).
  5. ITERATE with advanced pricing (Devex/steepest-edge), Harris ratio tests, and sparse basis updates.
  6. REFACTOR periodically or when growth/stability metrics exceed thresholds.
  7. POSTSOLVE map solution back to original model and reconstruct eliminated variables/rows.
  8. OPTIONAL if started from interior-point, run crossover to obtain a basic solution and exact basis status.

Mode Selection as a Mathematical Decision

The `Auto | Primal | Dual` choice in `simplex.h` is not just a coding convenience. It comes directly from which optimality conditions are already nearly satisfied.

Which Simplex Variant? $$x_B=B^{-1}b,\qquad \bar c_N=c_N-N^\top B^{-\top}c_B.$$ $$x_B\ge 0,\ \bar c_N\not\ge 0 \Rightarrow \text{primal simplex is natural},$$ $$x_B\not\ge 0,\ \bar c_N\ge 0 \Rightarrow \text{dual simplex is natural}.$$
  1. Primal simplex preserves primal feasibility and repairs dual infeasibility.
  2. Dual simplex preserves dual feasibility and repairs primal infeasibility.
  3. The `Auto` logic in the header mirrors this theorem-level distinction exactly.
  4. That is why a solver may switch to dual simplex after a bound/RHS/cut update even when the original solve used primal simplex.

Basis Update Engineering (What Actually Matters)

  1. Forrest-Tomlin / sparse LU updates: cheaper per pivot than full refactorization while preserving sparsity structure.
  2. Hyper-sparse solves: exploit very sparse right-hand sides in $Bd=A_e$ and $B^\top y=c_B$ for large-scale LPs.
  3. Markowitz-style pivoting in refactorization: balances sparsity and numerical stability during fresh LU builds.
  4. Refactor triggers: monitor update count, pivot growth, residual norms $\|Bx_B-b\|_\infty$, and dual residuals.
Concrete Implementation Signals from the Headers
  1. Adaptive pricers: the code distinguishes Devex, steepest-edge, partial pricing, dual steepest, and most-negative rules, then switches among them when degeneracy signals worsen.
  2. Bound-flipping controls: dual-simplex options such as `dual_allow_bound_flip` and pivot tolerances reflect the fact that modern dual simplex is not the naive classroom version.
  3. Refactor schedules: options like `refactor_every` and `compress_every` show that basis updates are monitored operationally, not left unchecked.
  4. Iterative refinement: the LU backend performs refinement after triangular solves, which is a concrete numerical-stability measure often omitted from theory notes.
  5. Non-destructive presolve modes: the presolver distinguishes safe reductions from structural eliminations, which matters when users need stable variable/constraint mappings.
Why the LU Backend Uses Markowitz, Rook Pivoting, and Refinement

The linear algebra in `simplex_lu.h` implements specific numerical decisions with clear mathematical purposes.

  1. Markowitz-style pivoting: when refactorizing $B$, choose pivots that approximately minimize $$\bigl(r_i-1\bigr)\bigl(c_j-1\bigr),$$ where $r_i$ and $c_j$ are the nonzero counts in the candidate row and column. This controls fill-in, so the factors stay sparse.
  2. Rook pivoting: do not accept a tiny pivot just because it is sparse. Search for an entry that is simultaneously large in its row and column, reducing growth and instability.
  3. Triangular solves instead of inversion: solve $$Lz=P_r b,\qquad Uw=z,\qquad x=P_c w,$$ rather than forming $B^{-1}$. This is cheaper and more stable.
  4. Iterative refinement: after computing a solve, form the residual $$r=b-Bx,$$ solve approximately for a correction $\Delta x$, and update $x\leftarrow x+\Delta x$. This reduces backward error caused by roundoff.
  5. Transposed solves matter equally: reduced costs require $$B^\top y=c_B,$$ so the LU backend must support both primal directions and dual multipliers with the same numerical quality.

Advanced Pricing and Ratio Tests

Harris Two-Pass Idea $$\theta=\min_{i:d_i>0}\frac{\bar b_i+\tau_p}{d_i} \quad\text{(pass 1 for feasibility tolerance),}$$ $$\text{then choose a numerically stable pivot among near-minimizers (pass 2).}$$
  1. Partial pricing / candidate lists: scan subsets of columns to reduce pricing cost per iteration.
  2. Devex resets: periodically reset weights to avoid stale curvature proxies.
  3. Bound-flipping ratio test (BFRT): especially in dual simplex, allows controlled bound flips to reduce stalling.
  4. EXPAND/perturbation variants: enlarge effective feasibility region slightly to prevent repeated degenerate pivots.
Pricing Theory Behind `simplex_aux.h`

The degeneracy manager and pricers in `simplex_aux.h` are implementing mathematically different approximations to the same question: which entering step gives the best improvement per unit geometric effort?

$$\text{Dantzig:}\qquad e=\arg\min_j \bar c_j,$$ $$\text{Steepest-edge:}\qquad e=\arg\min_j \frac{\bar c_j}{\|B^{-1}A_j\|_2},$$ $$\text{Devex:}\qquad e=\arg\min_j \frac{\bar c_j}{\sqrt{w_j}},\qquad w_j\approx \|B^{-1}A_j\|_2^2.$$
  1. Dantzig: cheapest to compute, but may zig-zag badly under degeneracy.
  2. Steepest-edge: best theoretical geometric normalization, but maintaining exact norms is expensive.
  3. Devex: a low-cost proxy for steepest-edge, which is why it appears so often in practical codes.
  4. Partial pricing: reduce pricing complexity from “scan every nonbasic column” to “scan a promising working set,” which matters when $n$ is huge.
  5. Degeneracy signals: the header tracks small steps, repeated entering columns, and objective stalling because those are exactly the observable symptoms of cycling risk.
Why Steepest-Edge Has a Geometric Foundation

Steepest-edge is not an arbitrary heuristic. It comes from measuring objective decrease relative to the actual edge direction in primal space.

  1. If nonbasic variable $x_j$ enters, the associated edge direction is $$\Delta x^{(j)}= \begin{bmatrix} -B^{-1}A_j\\ e_j \end{bmatrix},$$ where the first block is the change in basic variables and the second block is the entering nonbasic direction.
  2. The directional derivative of the objective along that edge is exactly the reduced cost: $$\frac{d}{d\theta}c^\top(x+\theta \Delta x^{(j)})=\bar c_j.$$ For a minimization problem, more negative $\bar c_j$ means faster immediate descent.
  3. But using only $\bar c_j$ ignores the length of the edge direction. A column with a very negative reduced cost may also produce a very large movement in the basis coordinates.
  4. If we normalize by the edge norm, the rate of decrease per unit Euclidean step becomes $$\frac{\bar c_j}{\|\Delta x^{(j)}\|_2} \approx \frac{\bar c_j}{\sqrt{1+\|B^{-1}A_j\|_2^2}}.$$ This is the geometric reason behind steepest-edge pricing.
  5. In most implementations the constant `1` in the norm is ignored or absorbed into the weight update, leading to the practical proxy $$\frac{\bar c_j}{\|B^{-1}A_j\|_2}.$$ That is the quantity appearing in the section above.
Why Devex Exists

Exact steepest-edge is attractive theoretically, but it is expensive because the weights $$\gamma_j=\|B^{-1}A_j\|_2^2$$ must be maintained for every nonbasic column.

  1. Computing all exact $\gamma_j$ values after every pivot would require too many solves or updates.
  2. Devex replaces exact norms by surrogate weights $w_j$ that are updated cheaply and reset periodically.
  3. The pricing rule becomes $$e=\arg\min_j \frac{\bar c_j}{\sqrt{w_j}},$$ which tries to preserve the geometric benefit of steepest-edge without paying its full linear-algebra cost.
  4. This is why production solvers often start with Devex, monitor behavior, and only escalate to more expensive pricing logic when degeneracy or instability becomes visible.
Why Degeneracy Management Needs State

The degeneracy manager is keeping statistical evidence about the pivot process, because anti-cycling in production is rarely one static rule.

  1. If many recent pivots satisfy $$\alpha \approx 0 \quad\text{or}\quad \Delta z \approx 0,$$ then the algorithm is stalling.
  2. A solver can respond by changing pricing strategy, rebuilding candidate pools, perturbing lexicographic order, or temporarily forbidding suspicious repeats.
  3. Mathematically, all of these are attempts to preserve the theorem-level invariants while biasing the trajectory away from degenerate cycles.
  4. This is why `simplex_aux.h` stores epochs, repeat windows, condition estimates, and performance windows: these are observable proxies for “the current pivot policy is no longer trustworthy.”

Presolve as Safe Model Equivalence

The presolver in `presolver.h` is not changing the optimization problem arbitrarily; it is applying equivalence-preserving transformations and recording them for postsolve.

Row Activity Bounds $$\min a_i^\top x,\ \max a_i^\top x \quad\text{computed from variable bounds } l\le x\le u.$$
  1. If a row's implied activity interval does not intersect its RHS/sense requirement, the model is infeasible before simplex starts.
  2. If a variable is fixed, substitute $x_j=\hat x_j$ and shift $$b\leftarrow b-A_j\hat x_j,\qquad c_0\leftarrow c_0+c_j\hat x_j.$$
  3. Scaling rows/columns changes coordinates, not the underlying LP; it is used to improve conditioning before basis factorization.
  4. The action stack in the header is the postsolve proof that every elimination can be reversed after the reduced LP is solved.
Certificates and Telemetry Are Part of the Mathematics

The solution object in `simplex.h` contains basis indices, status, iteration counts, and optionally a Farkas vector. That is not bookkeeping fluff.

  1. Optimality certificate: a basis plus $$A^\top y\le c,\qquad b^\top y=c^\top x$$ is a short proof of optimality.
  2. Infeasibility certificate: a Farkas vector $y$ with $$A^\top y\le 0,\qquad b^\top y>0$$ is a short proof that $Ax=b,\ x\ge 0$ has no solution.
  3. Telemetry: quantities like degeneracy streak, condition estimate, and rebuild count are operational diagnostics for how hard the theorem-level problem was numerically.
  4. A scientifically documented solver should expose these because they let users verify results and diagnose failures without reading the implementation internals.

Dual Simplex in Modern MIP/LP Engines

In branch-and-bound, each node changes bounds/rows slightly; dual simplex re-optimizes from the parent basis with very low warm-start cost. This is the dominant production strategy in many MILP solvers.

  1. Hot start basis transfer: inherit basis from parent node, then repair primal infeasibilities only.
  2. Node presolve: lightweight local reductions before re-optimization.
  3. Cut management interaction: newly added cuts typically break primal feasibility but preserve useful dual information.
  4. Backtracking/rebuild: if numeric quality degrades, rebuild factors and optionally re-crash.
Algorithm 12.2 — Dual Re-Optimization After Model Changes
  1. Start from previous basis $(\mathcal B, LU(B))$.
  2. Apply bound/RHS/cut updates; recompute $\bar b$ and keep $\bar c$ as reference.
  3. If dual infeasible, perform a short repair phase (pricing restricted to violating set).
  4. Run dual simplex iterations with BFRT/Harris safeguards.
  5. If stalling persists, trigger perturbation or switch pricing strategy (e.g., Devex $\leftrightarrow$ steepest-edge).
  6. On convergence, store basis as warm start for the next node or scenario.
Operational KPIs for a "Good" Simplex Run
  1. Low stall ratio (fraction of pivots with negligible objective change).
  2. Controlled refactor frequency (not too frequent, not too stale).
  3. Small primal/dual residuals at termination and stable basis condition indicators.
  4. Comparable primal and dual objective values within solver tolerance (duality gap near zero).
